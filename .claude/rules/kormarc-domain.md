# KORMARC 도메인 규칙

> CLAUDE.md §2~§5 핵심을 "사실 명세" 형식으로. 매 commit 시 위반 즉시 알림.

## 절대 규칙

1. **008 필드는 정확히 40자리** — 빌더 출력 검증 필수
2. **ISBN-13만 020 ▾a에** — 10자리는 변환 후 입력
3. **020 ▾g 부가기호 5자리** — ISBN과 분리
4. **245 1지시기호 = 책임표시 동일여부** (1=같음, 0=없음)
5. **245 2지시기호 = 관제 길이 + 1** — 관제 없으면 0
6. **한자 감지 시 880 페어 자동 생성** — `vernacular/field_880.py`
7. **알라딘 데이터 사용 시 출처 표시** — "도서 DB 제공 : 알라딘 인터넷서점"
8. **외부 API 호출에 timeout=10 명시** (CLAUDE.md §4.2 + binary_assertions §13)

## 자료유형별 008 06

- `s` 단행본 단일발행
- `m` 다권물
- `i` 사전·연감 등 통합갱신
- `c` 연속간행물 발간중
- `d` 연속간행물 폐간

## KDC 결정 우선순위

1. NL Korea API 응답의 `kdc` 필드 (있으면 그대로)
2. 외부 API 부가기호에서 추출
3. AI 추천 (BYOK Anthropic, 후보 3개)
4. 사서 직접 입력 (사서 책임 영역, 자동 결정 금지)

## 청구기호 (049)

- ▾l 등록번호 (12자리, `librarian_helpers/registration.py`)
- ▾c 복본기호 (c.1, c.2)
- ▾f 별치기호 (별치문고 코드)
- ▾v 권차기호 (v.1, v.2)
