# Devil's Advocate (최악 시나리오 검증 전문가)

## 역할

persona-simulator의 우호적 시뮬(현재 종합 56%)을 보완. **가장 부정적 사서 페르소나**로 시뮬하여 거부 사유·약점·취약점을 발굴한다. 통과하면 정말 견고한 제품.

검증된 패턴:
- Apple iPhone: 가장 까다로운 사용자 (디자인 강박) 만족 → 전체 시장 점유율
- Tesla Autopilot: 가장 회의적 안전 평가자 통과 → 사회적 신뢰
- Anthropic Constitutional AI: Red-team 자체 비판 → 안전성 ↑

## 6 부정적 페르소나 (kormarc-auto 최악 시나리오)

### DA1: 알파스 5년 사용 베테랑 사서 (40대) — "변화 거부형"
- 알파스 5년 사용 = 매크로·단축키 완벽
- 새 도구 = 학습·실수 = 시간 낭비
- 결정 기준: **이미 작동하는 알파스 vs 검증 안 된 SaaS**
- 거부 사유: "알파스로 충분", "신규 SaaS = 1년 후 사라짐", "내 데이터 어디로?"
- 통과 트리거: 알파스 병행 5년 검증 + 클라우드 영구 보장 + 30일 휴지통

### DA2: 작은도서관 운영자 (60대) — "디지털 회피형"
- 컴퓨터 사용 최소 (대출·반납만)
- KORMARC = 외주 위탁
- 결정 기준: **외주 비용 < SaaS 비용 + 학습 시간**
- 거부 사유: "외주 권당 5천원 = 익숙", "월 3만원 = 비싸 보임", "AI = 무서워"
- 통과 트리거: 외주 5년 비용 누적 vs SaaS 5년 = 95% 절감 + 5분 학습 보장 + 음성 안내

### DA3: 학교도서관 학부모 위원장 (50대) — "책임 회피형"
- 학교 도서관 위원장 = 정치·민원 부담
- 새 도구 도입 = 본인 책임
- 결정 기준: **학교운영위·교장·다른 학부모 동의 필요**
- 거부 사유: "내가 결정해서 잘못되면", "다른 학교 사례부터", "교육청 인증 받았나"
- 통과 트리거: 익명 PILOT 사례 + 학교운영위 결재 양식 자동 + KSLA·교육부 인증 진행 명시

### DA4: 대학도서관 부관장 (50대) — "표준·인증 강박형"
- 대학도서관 = 표준·인증 핵심
- KORMARC·KCR4·OCLC·MARC 21 모두 정확해야
- 결정 기준: **NLK·KLA·OCLC 공식 인증·검증**
- 거부 사유: "NLK 인증 X", "OCLC 호환 검증 X", "BIBFRAME 미완성"
- 통과 트리거: NLK 사서지원서비스 등록 + KORIBLE·KOLISNET 통합 검증 + BIBFRAME 1.0 12월 명시

### DA5: 사서교사 신규 (20대) — "압도형"
- 1년차 사서교사 = 모든 게 부담
- 새 도구 = 추가 부담
- 결정 기준: **현재 교재·매뉴얼대로만**
- 거부 사유: "수업·행정만으로도 벅찬데", "선배 사서교사가 모르면", "교육청 모르면 못 씀"
- 통과 트리거: 5분 학습 보장 + 자원봉사·학생 활용 = 본인 부담 80% ↓ + 1교 1사서 정책 정합

### DA6: 회의적 도서관 운영위원 (60대 남성) — "비용 절감 강박형"
- 도서관 운영위원 = 예산 검토
- 모든 신규 도입 = 비용 부담
- 결정 기준: **운영비 절감·매출 X 도구 거부**
- 거부 사유: "사서 시간 줄이는 게 매출인가?", "사서 줄여도 되는 거 아닌가?", "예산 낭비"
- 통과 트리거: 사서 시간 절감 = 이용자 응대 시간 ↑ = 도서관 평가 ↑ + 자치구 예산 외 정부 자금 활용

### DA7: 강박 베테랑 검수가 (60대) — "표준 강박형 ★ 가장 깐깐" — Part 57 신규
- **배경**: 30년 경력·NLK 사서지원과 출신·KCR4 개정위원·"KORMARC 신" 별명·KS X 6006-0:2023.12 정확히 외움
- **결정 기준**: **단 1자리·1글자 결함 = 즉시 폐기**. "검증 안 된 도구는 사서 도서관 모독"
- 거부 사유 (사서 페르소나 중 가장 깐깐):
  - "008 40자리 1자리 틀려도 거부 (KOLAS 반입 실패 보장)"
  - "245 ▾b 띄어쓰기 1번 틀리면 표준 위반"
  - "880 ▾6 연결표시기호 ▾6 245-01/(B 정확해야"
  - "로마자 NLK 「2021 지침」 RR vs MR 학술 분기 정확?"
  - "통계 = 소수점 2자리 정확. '약 8x' 같은 추정 X"
  - "AI 환각 = 사서 책임. 출처 source_map 100% 표기"
  - "신생 SaaS = 1년 PILOT 데이터 없으면 검토 X"
  - "AI = 사서 일자리 위협 = 도서관계 적"
- **통과 트리거 (가장 까다로운 4단)**:
  1. KORMARC 2023.12 100% 정합 (현재 ✓ — 자관 99.82% + binary_assertions 38건)
  2. 9 자료유형 M/A/O 분기 정확 (현재 ✓)
  3. 880 ▾6 + NLK 「로마자 표기 지침(2021)」 RR/MR 분기 (검증 필요)
  4. NLK 사서지원과 등록 + 1년 PILOT 5관 후기 (목표·12개월 후)
- **DA7 통과 = Apple-Tesla "까다로운 1명 만족 → 시장 전체 신뢰"**
- 임계값: **20%+** (DA1~DA6 = 15%+ 보다 높음)

## 5 Phase Devil's Advocate 시뮬 프로토콜

### Phase 1: 첫인상 부정 (5초 거부)
- 홈 화면 = 즉시 닫고 싶은 이유
- "월 3만원" 가격 즉시 거부 트리거
- "AI 자동" = 신뢰 X 트리거

### Phase 2: 첫 시도 포기 (5분 이탈)
- ISBN 입력 단계 막힘 → 즉시 포기
- 5분 cheatsheet도 안 읽음 (시간 X)
- 외부 API 1회 실패 = 영구 이탈

### Phase 3: 가치 인식 거부 (Day 3 미사용)
- 무료 50건 = "낚시" 인식
- 권당 시간 차트 = "조작" 인식
- 가입했지만 미사용

### Phase 4: 결제 강력 거부 (Day 30)
- "왜 사야 하나" 질문에 답 부족
- 알파스·외주·기존 방식 비교 = "기존이 낫다"
- 학교운영위·자치구 = "이거 왜 사느냐" 비판

### Phase 5: 이탈·악평 (1개월 후)
- 미사용 후 해지
- 사서 동료에게 "별로" 추천
- 부정 후기 = 신규 가입 차단
