## 3. 카탈로깅 워크플로우 — KOLAS 25년 사용자

### 3.1 KOLAS 사용 패턴

- **신간 등록 권당 시간**: 5~7분 (숙련자, KORMARC 18~22필드 수동 입력)
- 분류기호 결정에 **2~3분 추가** (애매한 책의 경우)
- 외주 마크 받은 후 검수: 권당 2~4분
- KOLAS 시스템 다운 시 멘탈 붕괴

### 3.2 최영자의 자부심

- "25년 KORMARC 작성한 손과 눈이 책을 보면 분류가 보인다"
- "AI가 KORMARC 자동 생성한다고? 8필드 정도는 가능할지 몰라도, 045·245$d·505·520·6XX는 사서가 직접 봐야 한다"
- "외주 마크 품질 낮은 거 다 사서들이 손봐서 살리는 거다"

### 3.3 최영자가 아는 진짜 어려움

- KDC 분류 애매한 학제간 주제 (사회과학+역사 등)
- 동양고서 분류 (KORMARC 고서용 별도)
- 다국어 자료 (영어·중국어·일본어 혼재)
- 시리즈물·연속간행물 (CR) 등록의 복잡성

→ 최영자가 보기엔 kormarc-auto가 잘하는 영역(ISBN 자동 생성)은 **이미 사서가 외주로 해결하고 있어서 새 도구 필요성이 낮음**.

## 5. 의사결정 구조 — 영업 시 무시해도 됨

```
최영자 (사용자, 거부자)
    ↓
도서관장 (정규교원 또는 행정직 보직, 사서직 다수)
    ↓
지자체 (시·군·구청 문화체육과)
    ↓
시도지자체 광역도서관 정책 (KOLAS 국책 사업이라 별도 시스템 도입 어려움)
```

**최영자의 영향력**:
- 본인이 도구 도입 결정 못 함, 그러나 **반대 의견 强**하면 도서관장 반려 가능
- 후배 사서 5~10명에게 영향력 (선임 사서)
- 사서 협회·노조에서 발언권

## 6. 거부 페르소나의 진짜 우려 — 사전 대응이 결정적

### 6.1 우려 1: "사서 전문성 위협"

**대응 메시지** (모든 마케팅 자료에 명시):
> "kormarc-auto는 사서의 전문성을 강화하는 도구입니다. AI는 12~15필드 초안만 생성하고, 마지막 결정은 사서가 합니다. KDC 분류, 주기, 책임표시 등 사서의 판단이 필수입니다."

### 6.2 우려 2: "사서 일자리 위협"

**대응 메시지**:
> "kormarc-auto는 외주 마크 시장을 흡수합니다. 사서 일자리를 줄이는 게 아니라, 외주가 가져간 일을 사서에게 돌려주는 도구입니다. 사서가 직접 검수하면서 시간은 단축됩니다."

### 6.3 우려 3: "KOLAS·DLS 대체"

**대응 메시지**:
> "KOLAS·DLS는 그대로 쓰세요. kormarc-auto는 .mrc 파일만 출력하고, 일괄 import는 KOLAS·DLS에서 진행합니다."

### 6.4 우려 4: "AI의 부정확한 KDC 분류"

**대응 메시지**:
> "KDC 자동분류는 류·강·목 (3자리) 약 85~90% 정확도, 사서 검수 후 사용. 첫 단계 초안일 뿐입니다."

### 6.5 우려 5: "외주 사서 직업 잠식"

**대응 메시지**:
> "외주 마크 작업자 (지역서점, 마크 전문업체)는 다른 부가가치 작업으로 전환 가능. kormarc-auto는 도서관 사서·외주 사서 양쪽 모두에게 활용 가능한 도구."

## 7. 최영자 페르소나가 영업 funnel에 들어왔을 때

영업이 잘못 진행되어 최영자에게 콜드 콘택트한 경우:

### 첫 미팅 (15~30분)
- 거부 의견 그대로 청취 (반박 X)
- 우려 5가지 인정 ("선생님 말씀이 맞습니다, 25년 경험은 AI가 대체 못 하죠")
- **본관 KOLAS 그대로 + 분관 또는 신참 사서용 보조 도구로만 제안**
- 분관·신참 사서 시범 PILOT (최영자는 사용 X)

### 거부 페르소나의 가치
- **반론 listing**: 마케팅 자료에 "KOLAS 25년 사용자가 우려하는 것" 별도 섹션
- **세일즈 대본 strengthening**: 영업 직원 교육 시 최영자 페르소나 시뮬레이션 필수

## 8. 최영자 페르소나가 잠재적 Champion으로 전환되는 시나리오 (저확률)

만약 다음 조건이 충족되면 최영자도 사용 가능:

1. **본인 부서가 아닌 분관·신참 사서가 1년 사용** → 품질·시간 입증
2. **전국 사서 학회·협회에서 검증된 사례 5건 이상**
3. **KORMARC 정합성 자체 검증 도구** (사서가 직접 검증 가능한 모드)
4. **사서가 만든 도구라는 거버넌스** (한국도서관협회 협력 등)
5. **KDC 자동분류 정확도 90% 이상 입증**

→ 1~2년 후 가능. 단 v2 전략의 1년차에는 무시.

## 9. v2 전략에서 최영자의 역할 — 거부 페르소나 활용

### 9.1 마케팅 자료의 별도 섹션
- "KOLAS 25년 사용자의 우려, 어떻게 대응하나" FAQ 5개 (위 §6 참조)

### 9.2 세일즈 대본 시뮬레이션
- 영업 직원이 최영자 페르소나 vs 김지원 페르소나 양쪽 대본 학습
- 거부 시 즉각 후퇴, 본관 LMS 그대로 + 보조 도구 메시지로 전환

### 9.3 사서노조·협회 채널
- 한국도서관협회·전교조 사서교사위원회·전국사서교사노조에 사전 설명
- "AI가 사서 대체"가 아닌 "AI가 사서 권한 강화" 메시지 일관

### 9.4 거부 페르소나 인터뷰 1편
- 학교도서관저널 또는 도서관문화 (한국도서관협회 회보)에 비판적 시각 인터뷰 게재
- "AI 도구를 잘 쓰는 방법" 논의 → 권위 있는 사서가 검증

## 11. 거부 페르소나 시뮬레이션 — 영업 직원 교육 대본

### 시뮬레이션 1: 첫 콜드 콜

```
영업: "안녕하세요 선생님, kormarc-auto에서 연락드렸습니다."

최영자: "또 AI 도구네요. KOLAS 25년 썼습니다. 외주 마크도
        사서가 다 손봐요. 새 도구 필요 없어요."

영업: "선생님 말씀이 맞습니다. 25년 경험은 어떤 도구도
       대체 못 하죠. 저희는 사서 선생님의 시간을 외주 검수에서
       다른 업무로 옮기는 도구일 뿐입니다."

최영자: "다른 업무라니. 사서 일자리 줄이려는 거 아니에요?"

영업: "오히려 외주가 가져간 일을 사서에게 돌려주는 도구입니다.
       외주 권당 평균 237원, 저희는 200원이고 사서가 직접 검수
       하면서 작업합니다. 사서 일자리 가치를 올리는 도구입니다."

최영자: "그래도 KOLAS 안 바꿉니다."

영업: "KOLAS 그대로 쓰세요. 저희는 .mrc 파일만 출력합니다.
       KOLAS 일괄 import는 선생님이 하시는 그대로입니다.
       부담스러우시면 분관 신참 사서 1명만 시범 사용해보시고,
       1년 후에 평가만 해주셔도 됩니다."

최영자: "...일단 알겠습니다. 자료만 보내주세요."
```

## 13. 결론 — 최영자 페르소나는 영업 ICP 제외, 그러나 마케팅·거버넌스에 필수

- **영업 우선순위**: 김지원 > 박서연 > 정유진 > 이민재 >> 최영자
- **마케팅 메시지 검증**: 모든 자료를 최영자 페르소나가 비판적으로 검토했을 때도 신뢰성 있는지 검증
- **거버넌스 참여 초청**: 한국도서관협회 협력 사업 참여, 사서노조 분기별 미팅
- **1~2년 후 재방문**: PILOT 5곳 사례 확보 후 최영자 페르소나에게 다시 접근, 분관·신참 사서 시범 사용 제안
