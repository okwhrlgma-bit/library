## TLDR (한 줄 요약)

한국 사서가 일상에서 쓰는 시스템은 **40여 종**(자료관리·통합검색·부가서비스·학술DB·통계·RFID 등)에 걸쳐 있지만, kormarc-auto가 실제로 **흡수해야 할 곳은 5개(KOLAS III · 독서로DLS · KOLIS-NET · 정보나루 · 알파스/MARCXML)뿐**이고 나머지는 **간접 연동(파일 import) 또는 무시**가 정답이다. 핵심 페인포인트는 "옛날 UI · 폰 미지원 · Excel 의존 · 권한 절차"이며, 우리는 **"KOLAS의 마크 반입 폴더에 .mrc 떨궈주기 + 사서 폰 브라우저"** 로지컬에 집중하면 된다. 2026-12-31 KOLAS III 표준형 기술지원 종료라는 기회/위협을 동시에 안고 있다.

## 3. 연동 설계 권고 (4가지 채널 평가)

| 채널 | 적합도 | 이유 | 우리 채택 |
|---|---|---|---|
| **A. 파일 import/export** (.mrc, MARCXML, CSV) | ★★★★★ | KOLAS·DLS·알파스·TULIP 모두 지원, 권한 무관, 가장 안전 | ✅ **메인** |
| **B. REST API 직접 호출** | ★★ | KOLAS·DLS는 미지원 / 정보나루·KOLIS-NET·KCI만 지원 | ✅ **수신만** (조회 폴백) |
| **C. 브라우저 확장(Chrome)** | ★★★ | KOLAS는 데스크톱 앱이라 무관, **알라딘·교보·정보나루 페이지에서 ISBN 우클릭 → kormarc-auto 보내기**는 강력 | 🟡 **Phase 2** |
| **D. Excel·HWP 자동화** | ★★★★ | 사서가 매일 쓰는 도구, 신착·통계·라벨 양식 | ✅ **이미 일부**(.xlsx, 라벨), HWP는 보류 |

**핵심 원칙**: "사서 PC에서 .mrc 파일 → KOLAS 자동 반입 폴더 떨굼" 한 흐름이 90% 가치. 나머지는 보조.

## 4. 우리 로드맵 제안 (Phase별)

### Phase 1 (지금 ~ 6개월) — **사서 개인 도구로 완성**
- ✅ KOLAS III · DLS · MARCXML · CSV (이미 구현, v0.4.1)
- ✅ KOLIS-NET·정보나루·NL Korea ISBN 폴백 (이미 구현)
- ✅ Streamlit 모바일 (이미 구현, v0.3)
- 🆕 **KOLAS 2026-12-31 지원 종료 대비 영업 메시지** 작성 → `docs/sales-roadmap.md`에 반영
- 🆕 책단비 띠지·신착 간지 라벨 옵션 → `output/labels.py`

### Phase 2 (6 ~ 12개월) — **부가 서비스 간접 연계 + 파트너십**
- 🟡 **국가도서관통계 자동 보고서** (libsta 양식 매핑) → `output/reports.py` 확장
- 🟡 **KCI·RISS 학위논문 OpenAPI 폴백** → 502 검증
- 🟡 **Chrome 확장 PoC** — 알라딘·교보·정보나루 페이지에서 ISBN 우클릭 → kormarc-auto 전송
- 🟡 이씨오·북이즈·제이넷 **재판매 파트너십** 시도 (수수료 20~30%)
- 🟡 KLA·KSLA·사서 카페 마케팅 채널화

### Phase 3 (12 ~ 24개월) — **차세대 KOLAS 전환기 진입**
- 🟡 KOLAS III 후속 시스템 발표 시 **즉시 호환 모듈** 개발
- 🟡 책나래(점자·녹음·대형활자) 라벨·필드 강화
- 🟡 책바다 자관 인덱스 연계 (`inventory/library_db.py`)
- 🔴 책이음·DBpia·KISS·키오스크는 **계속 무시** (도메인·라이선스 부적합)

### 영구 무시 (헌법 준수)
- 모든 RFID 하드웨어 / 자가대출반납기 / 게이트 / 학술 DB 본문 / 회원 PII / KOLAS 소스코드 직접 분석.

## 5. 결론 (PO 한 줄 의사결정 카드)

> **"KOLAS·DLS·MARCXML 파일 출력 + 정보나루·KOLIS-NET 조회"** 5축에만 집중. 나머지 35종은 사서가 알아서 쓰게 두고, 우리는 **2026-12-31 KOLAS III 종료**가 만드는 시장 변화에 가장 빨리 적응할 수 있는 위치를 점한다. Chrome 확장과 libsta 자동 보고서가 다음 두 개의 큰 수익 레버.
