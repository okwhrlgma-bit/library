| 긴급대출 | ✅ | ❌ |
| 책밴드 (자체 상호대차) | ✅ | 🟡 csv export |
| 책이음 (전국 통합) | ✅ (호환) | 🟡 양식 자동 |

## 1. 알파스 §6.1 등록자료관리 (p.56~63)

### 7 핵심 작업

| 작업 | 메뉴 | 우리 정합 |
|---|---|---|
| 등록자료조회 | `등록 > 등록자료관리 > 기본화면` | ✅ |
| 상세보기 | `등록자료관리 > 기본화면 > 상세보기` | ✅ |
| 입력/수정/삭제 | `등록자료관리 > 기본화면 > 툴바 > 입력·수정·삭제` | ✅ KORMARC 표준 |
| **등록번호부여 (일괄)** | `등록자료관리 > 기본화면 > 툴바 > 등록번호부여` | ✅ KORMARC 049 ▾l |
| **누락번호부여** | `등록자료관리 > 기본화면 > 툴바 > 등록번호부여 > 누락번호적용` | 🟡 우리 신규 후보 (자관 누락 등록번호 회수) |
| 번호취소 | `등록자료관리 > 기본화면 > 툴바 > 번호취소` | 🟡 우리 신규 (취소된 번호 → 누락등록번호로 등록) |
| **검수취소** | `등록자료관리 > 기본화면 > 툴바 > 검수취소` | ❌ **신규 모듈 후보** (재검수 워크플로우) |
| 정리대상인계 | `등록자료관리 > 기본화면 > 툴바 > 정리대상` | ✅ → `목록완성` (우리 `kormarc/builder.py` 통합) |
| 완료처리 | `등록자료관리 > 기본화면 > 툴바 > 완료처리` | 🟡 부분 (희망자료 자동 SMS는 우리 영역 X) |

### 자료 반입 3 형식

| 형식 | 우리 정합 |

### 검색 조건 매핑

| 알파스 검색 조건 | 우리 SaaS |
|---|---|
| 수입년도·수입구분 (구입/기증/적용안함) | ✅ |
| 차수번호 | 🟡 |
| 자료상태 (등록인계자료·등록자료) | 🟡 |
| **원부년도·원부번호** | ❌ **신규 모듈** (원부대장관리) |
| 등록번호 | ✅ |
| 인계처리일 | 🟡 |

### 등록번호 일괄·누락 워크플로우 (p.58~61)

```
1. 등록번호 확인 화면
   - 선정 자료의 등록구분에 따라 마지막 번호 다음 번호부터 자동 부여
2. 원부번호 확인 화면
   - 등록원부 마지막번호 다음 번호 (관리메뉴 설정)
3. 마지막번호관리 화면
   - 자동 부여되는 마지막 등록번호·원부번호 검색·수정
4. 누락번호적용 화면
   - 이전 삭제 또는 누락등록번호로 입력한 번호 부여
```

→ **자관 등록번호 prefix `EQ`** (D 드라이브 발견) = 알파스 등록번호부여 시 자동 prefix 정합.

## 3. 알파스 공통 §1~§3 핵심 (재정리)

### §1 개요

| 항목 | 정보 |
|---|---|
| 적용 | 공공도서관·대학도서관·작은 도서관 |
| 단독관 vs 통합관 | 통합관 = 분관 + 작은도서관 포함 + 도서관간 상호대차 + RFID |
| 8 제공 업무 | 비치희망도서·구입·기증·주문·검수·등록·정리(목록)·배가·열람 |
| 설치 | 제조사(이씨오) 전담 엔지니어 — 사용자 설치 X |

### §2 알아두기

| 항목 | 정보 |
|---|---|
| 비밀번호 정책 | **10자 이상 + 영문/숫자 또는 영문/숫자/특수문자 [!@#$%^&*()] 혼용** |
| 대시보드 | 휴관일·빠른이동·공지사항·대출/반납 현황 |

## 5. 영업 메시지 (알파스 사용 도서관 대상)

> "알파스 사용 도서관 사서들이 매일 [구입] → [검수] → [등록자료관리] → [등록번호부여] → [정리대상] → [목록완성] 6단계를 진행하는 시간을, 우리 SaaS는 ISBN 입력 1번으로:
>
> - KOLIS-NET·SeoJi·Aladin 4단 폴백 자동 검색
> - KORMARC 표준 자동 생성 (245·260·300·056·049 등)
> - 4단 검증 (구조·스키마·Indicator·문자/길이)
> - 알파스 import 양식 (엑셀·한셀·MARC 3 형식) 자동 export
>
> 알파스의 검수취소·누락번호부여·원부대장관리 등 사서 routine을 우리 SaaS가 자동 트리거합니다."
