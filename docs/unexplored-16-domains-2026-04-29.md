## 1. ★ ROI 1순위 5 영역 (Q1 ≥ 80)

### 1.1 C1 장서 점검 (재고조사) — **법적 의무**

```
공공도서관 정기 장서 점검 의무 (도서관법)
- 전체 장서 목록 출력 (KOLAS CSV)
- 서가별 바코드 스캔
- 목록 대조 → 분실·오배가·파손
- 결과 보고서 (문체부 제출용)
- 분실 KORMARC 상태 자동 갱신

우리 SaaS:
- inventory/inspection.py 신규 (이미 부분 — 강화)
- KDC 순·서가 순 점검 목록 자동 생성
- 스캔 결과 자동 대조 + 차이 표시
- 문체부 제출 양식 자동
```

**Q1=90·연간 1~2회·법적 의무 = 결제 트리거**.

## 5. 도서관마다 다른 부분 정합 (정책 ③ 일반화)

PO 명령 정합: "도서관마다 다른 부분도 있을 것 — 관련 조사 및 적용 필요"

### 16 영역 모두 자관별 변형 가능

```yaml
# config.yaml — 16 영역 자관별 정책
unexplored_modules:
  inspection:
    enabled: true
    schedule: "annual"  # 자관별 (연간·반기·분기)
    target_section: ["KDC", "shelf"]
    submission_format: "moetcsv"   # 문체부 양식 자관별

  programs:
    enabled: true
    types: ["반기", "노인", "다문화"]   # 자관 운영 유형
    grant_voucher: false   # 강사비 지급 양식 자관별

  recommendation:
    enabled: false   # PIPA 옵션 C 진입 시
    algorithm: "collaborative_filtering"

  budget_roi:
    enabled: true
    fiscal_year_start: "2026-01-01"
    threshold_low: 0.15   # 잔여 예산 15% 이하 시 알림

  disposal:
    enabled: true
    age_threshold_general: 10   # 발행년+10
    age_threshold_law: 5         # 법령집 5년
    loan_count_threshold: 0      # 대출 0회

  donation:
    enabled: true
    confirmation_form_template: null   # 자관 양식
```

→ ADR 0022 양식 resolver 4단 fallback 정합.
