# 사서 업무 종합 가이드

> 사서 출신 PO에겐 익숙한 영역이지만 **개발자·신규 입문자가 전체 그림을 잡을 때** 사용.
> 우리 코드와 문서가 어디 사서 업무를 직접 도울 수 있는지 명시.

---

## 1. 사서 업무 분류 (한국 도서관계 기준)

### 1.1 정보봉사 (이용자 직접 응대)
- 참고봉사 (상호대차 포함)
- 독서지도·독서치료
- 어린이·청소년 프로그램
- 평생교육·문화행사 운영
- 정보활용교육 (information literacy)

### 1.2 자료조직 ★ 우리 도구 핵심 영역
- **수서**(收書): 도서 선정·구입·기증접수
- **분류**: KDC·DDC·LCC 등 분류기호 부여
- **목록**: KORMARC·MARC21 서지레코드 작성 (245·100·260…)
- **주제명**: NLSH·LCSH 주제어 부여
- **장서점검**(inventory): 연 1~2회 전수조사
- **제적**(除籍): 손상·분실·낡은 자료 폐기

### 1.3 자료관리
- 등록(accessioning) → 등록번호 부여
- 청구기호 부여 → 라벨 인쇄
- 배가(shelving)
- 보존서고·서가 관리

### 1.4 행정
- 예산·통계·연간보고서
- 인사·자원봉사 관리
- 시설 관리
- 도서관 운영위원회 회의

## 2. 사서 일상 업무 흐름 (신착도서 처리)

```
[1] 도서 입수 (구입·기증)
    ↓
[2] 등록(accession): 등록번호 부여, 입수처 기록
    ↓
[3] 검색: 정보나루·국립중앙도서관에 ISBN 조회
    ↓ (있으면)
[4a] 서지 다운로드 (KORMARC) → 자관 양식에 맞게 수정
    ↓ (없으면)
[4b] 표지·판권지 보고 직접 입력 (245·100·260·300·504·505·520)
    ↓
[5] 분류기호 부여 (KDC 또는 DDC) — 사서 책임 영역
    ↓
[6] 주제명 부여 (NLSH 또는 자체 어휘)
    ↓
[7] 청구기호 부여 (049): 분류기호 + 저자기호 + 권차/복본
    ↓
[8] 라벨 인쇄·부착 (책등용 청구기호 + 안쪽 바코드)
    ↓
[9] 데이터 저장 (KOLAS·DLS·KOLASYS 등 자관 시스템)
    ↓
[10] 배가 → 서가에 꽂음
```

**우리 도구 매칭**:
- [3]·[4a] ✅ ISBN 검색 + 외부 API 통합
- [4b] ✅ Vision (사진) + OCR (AI 0원)
- [5]·[6] ✅ KDC AI 후보 + NLSH AI 추천 (BYOK)
- [7] ✅ `librarian_helpers/call_number.py`
- [8] ✅ `output/labels.py` (PDF, A4 Avery)
- [9] ✅ KOLAS·DLS·MARCXML·CSV 출력

## 3. 사서 업무의 결과물 (아웃풋)

| 아웃풋 | 형식 | 누가 봄 |
|---|---|---|
| 서지레코드 | KORMARC `.mrc` 파일 | 카탈로그 시스템·이용자 검색 |

## 4. 사서 업무에서 힘든 부분 (페인포인트)

### 4.1 시간 압박
- 신착도서 처리: 권당 **8~15분** (분류·청구기호·라벨)
- 1인 도서관: 본업 + 마크 작업 + 행사 운영 동시
- 학교도서관: 사서 1명이 학생 응대까지 → 야근 흔함

### 4.2 표준 변동성
- KCR4 → KCR5 전환 진행 중 (2024.5 초안)
- DLS 개편 (2024 "독서로DLS" 통합)
- 새 자료유형 출현 (전자책·오디오북·점자도서)

### 4.3 책임 부담
- 잘못된 분류 → 자료 검색 안 됨 → 도서관 신뢰도 추락
- 마크 데이터 손실 → 5년 작업 날아감
- KOLAS 반입 거부 → 야근

### 4.4 도구 한계
- KOLASYS-NET·DLS는 무료지만 UI 구식
- 상용 도구(LAS·KOMET) 비싸고 종속
- 모바일 미지원
- 옛 책·기증도서 자동화 도구 부재

### 4.5 인력 문제
- 사서 1명이 모든 자료유형 처리 (도서·DVD·점자·전자책)
- 자원봉사자·사회복무요원 1차 입력 → 검수 부담

### 4.6 외부 의존
- 정보나루 API 인증키 1~3일 대기
- 알라딘 API 블로그 등록 필요
- KDC 분류표 종이책 옆에 두고 작업

## 6. 사서 종류 (자격·역할별)

### 6.1 자격
- **사서**(2급): 4년제 문헌정보학과 또는 전문대 + 실무 360시간
- **사서**(1급): 사서 자격 + 5년 이상 실무
- **정사서**: 임용 후 정규직
- **계약직 사서**: 작은도서관·학교 자원봉사 형태도 多

### 6.2 근무처
- 공공도서관 (시·도립)
- 대학도서관
- 학교도서관 (초·중·고)
- 작은도서관 (1,840곳, 마을·사찰·교회)
- 특수도서관 (법학·의학·기업·국회·법원·종교·박물관)
- 국가기관 (국립중앙도서관·국회도서관)

### 6.3 우리 ICP 우선순위
1. **1인 사서 학교도서관** (시간 압박 최대, 결제 의향 높음)
2. **신규 작은도서관** (마크 인력 부재, 무료 대안 한계)
3. **출판사** (납본·기증용 KORMARC 자체 생성)
4. **특수도서관** (사실검증 보고서 권고: 지불 확률 65%)
5. 대학도서관 (우리 ICP 외, MVP-2 고려)

## 9. 사서·도서관 미래 동향 (2024~2026)

| 트렌드 | 영향 |
|---|---|
| 생성형 AI 도입 (국회도서관 보고서 2024.12) | 자동 분류·서지 생성 80% 정확도 도달 |
| BIBFRAME 전환 검토 | MARC 대체 표준, 미국 LC 주도 |
| FRBR·RDA 확산 | 개체-관계 모델 |
| 전자책·오디오북 비중 증가 | 자료유형 분기 필요 (`material_type.py`) |
| 사서 인력 감소 | 자동화 도구 수요 ↑ |
| 작은도서관 무료 대안(KOLASYS-NET) | 우리 차별화 = 사진+AI+모바일 |

## 10. 우리 PO의 강점 활용

- ✅ 사서 출신 → "사서가 만든 도구" 브랜드
- ✅ 도서관 인맥 → 베타 사서 5명 모집 가능
- ✅ 한국 도서관 표준 깊이 이해 → KOLAS·DLS·KOLASYS 호환 설계
- ✅ 사서 페인포인트 직접 경험 → 진짜 차별화

이건 **다른 일반 개발자가 절대 못 따라오는 영역**. 영업·콘텐츠·제품 결정에 모두 활용.
