# SLA — Service Level Agreement (서비스 수준 약정)

> kormarc-auto · v1.0 · 2026-05-03

---

## 1. 가용성 (Uptime)

| 등급 | 목표 | 월 다운타임 한도 |
|------|----|----|
| Starter (B2B) | 99.5% | 3.6시간 |
| Pro (B2B) | 99.9% | 43분 |
| Enterprise (자치구 25관+) | 99.95% | 22분 |

### 측정·보고
- UptimeRobot 또는 동급 도구 (T3 DevOps)
- 월간 SLA 보고서 자동 (DT2)
- 이용자에게 분기 공개

---

## 2. 응답 시간 (Performance)

| 동작 | 목표 (95th percentile) |
|------|----|
| ISBN → KORMARC 결과 | ≤3초 |
| 검색 (자관 인덱스) | ≤500ms |
| 일괄 처리 1만 권 | ≤30초 |
| 사이드바 로딩 | ≤1초 |

---

## 3. 데이터 보존·복구

| 항목 | 보장 |
|------|----|
| 자동 백업 | 매일 03:00 KST |
| 백업 보관 | 30일 (rolling) |
| 분기 백업 | 1년 보관 |
| 복구 시간 (RTO) | 4시간 |
| 복구 시점 (RPO) | 24시간 (전일 백업) |
| 데이터 export | 1초 (TXT·MARC·CSV·MARCXML·MODS·JSON·OAI-PMH) |

| 사건 | 신고 시간 |
|------|----|
| PIPA 위반 (5만 명 이상·민감정보·해킹) | 72시간 (PIPA §39의4) |

## 5. 위반 시 보상 크레딧

월간 SLA 미달 시 다음 달 청구액에서 자동 차감:

| 가용성 | 크레딧 |
|------|----|
| 99.5% 미만 (Starter) | 다음 달 10% |
| 99.0% 미만 | 다음 달 25% |
| 95.0% 미만 | 다음 달 50% |
| 90.0% 미만 | 다음 달 100% (전액) |

---

## 6. 정기 유지보수

- 사전 공지: 7일 전 이메일·인앱 알림
- 시간: 매월 첫째 일요일 02:00~04:00 KST (도서관 폐관 시간)
- 다운타임 = SLA 측정 X

---

## 7. 폐업·서비스 종료 시

- 6개월 사전 공지
- 데이터 export 무료 보장
- 6개월 보존 (에스크로)
- 환불: 잔여 기간 일할 계산
