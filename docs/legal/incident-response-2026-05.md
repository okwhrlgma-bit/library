# 침해사고 대응 매뉴얼 (kormarc-auto SaaS) — 2026-05-03

> PIPA 2026-09-11 시행 (과징금 매출 **3% → 10%**) 대응.
> 카카오 학습 (151억 과징금): "부분 적용 후 기존 데이터 미마이그레이션 = 결정타".

---

## 1. 침해사고 정의 (PIPA §29-2)

다음 중 1건이라도 발생 시 = 침해사고:

1. **개인정보 유출** (USB·DB 덤프·로그 파일·스크린샷 포함)
2. **무단 접근** (인증 우회·권한 상승·SQL injection·세션 탈취)
3. **무단 변조** (DB 직접 조작·MITM)
4. **무단 삭제** (악의적 DELETE·랜섬웨어)
5. **시스템 침입** (RCE·SSRF·LFI)

---

## 2. 24시간 통보 의무 (PIPA §34)

### 발동 조건
다음 중 **1건이라도 충족** 시 24h 내 통보 의무:
- 1,000명 이상 정보주체
- 민감정보·고유식별정보 포함
- 해킹 등 외부 침입

→ kormarc-auto = B2B SaaS·회원 < 1,000명 단계는 의무 X·**자율 통보 권장** (신뢰).

### 통보 대상
1. **개인정보보호위원회** (privacy.go.kr) — 공식 신고
2. **한국인터넷진흥원 (KISA)** (privacy.kisa.or.kr·118) — 기술 지원
3. **정보주체** (영향 받은 회원) — 이메일 + 서비스 내 공지

## 7. 시스템 안전 조치 (사전 예방)

### 기술적 (코드)
- bcrypt cost 12+
- TLS 1.2+ 강제 (HSTS)
- AES-256 at-rest
- audit_log + 해시 체인 (변조 불가 증명)
- `incident_logger` 모듈 (`src/kormarc_auto/safety/incident_logger.py`)

### 인프라
- Cloudflare WAF (DDoS·SQL injection 차단)
- AWS / Cloudflare 백업 (Litestream → MinIO 10초)
- Bugsink Sentry 자동 에러 수집
- Langfuse LLM 비용·요청 추적

### 운영
- 분기별 침투 시뮬 (자가 테스트)
- M5 멀티테넌시 격리 (도서관 A ↔ B 누수 0)
- 1인 SaaS = PO가 보안 책임자 (이메일 + SMS 알림 24h)
