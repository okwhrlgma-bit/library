## 1. KOLAS 단행자료 워크플로우 (교육자료 핵심)

### 단행 메뉴 4단계

```
수서 → 등록 → 정리 → 배가
```

### 5 핵심 모듈 (교육자료 p.7~46)

| 모듈 | 메뉴 | 핵심 기능 |
|---|---|---|
| **수서** | 희망자료관리 | 비치희망자료 입력·검색 (이용자 신청) |
| **수서** | 구입자료관리 | MARC반입·국가자료공동목록반입·복본조사·차수번호 |
| **수서** | 기증자료관리 | 기증자관리·기증자료보기 |
| **등록** | 등록자료관리 | 코드변경·정리대상 (서지정보 수정·등록번호) |
| **등록** | 원부대장관리 | 원부추가·원부번호수정 |
| **정리** | 목록완성 | MARC 정리·종분리·부록자료·일괄변경 |
| **정리** | 부록정보 | 부록 등록번호 자동/수동 부여 |
| **열람** | 배가자료관리 | 배가변경·코드변경·이용제한구분·수입구분 |
| **열람** | 일괄변경 | 별치기호 일괄 부여 |

### 자료 반입 (KOLIS-NET)

- KOLIS-NET = 국가자료공동목록 (NLK 운영)
- 메뉴: `[단행 > 정리 > 목록완성 > 입력 > 반입 > 국가자료공동목록반입]`
- 자료형태·서명·저자명 검색 → MARC 반입

## 2. 책두레 5단계 워크플로우 (14p 가이드 정독)

```
1. 신청 (이용자 또는 사서)
   - KOLAS III: [공통]-[상호대차]-[책두레]-[책두레] 메뉴
   - 옵션 ON 시 자동 요청 (사서 개입 X)
2. 요청처리 (요청도서관)
3. 발송 (제공도서관)
   - 요청거절·요청취소·발송거절·발송취소·SMS 자동 발송
4. 입수 (요청도서관)
   - 입수 시 SMS 자동 발송 (이용자 알림)
5. 책두레 대출 → 반납
```

## 3. KOLAS III 자주 묻는 질문 (FAQ) — 우리 SaaS 함의

### FAQ 1 — 비치희망자료 우선대출예약

> "[희망자료관리] 메뉴에서 설정한 우선대출예약 기능은 마크를 반입하여 자료를 입력할 경우, 기능하지 않습니다. 반드시 [희망자료관리] → 등록인계 → 열람인계 → 배가일자기록 처리되어야 우선대출예약이 가능합니다."

→ **함의**: KOLAS III 워크플로우가 복잡 = 우리 SaaS UI에서 명확히 분기 안내 필요.

### FAQ 2 — 부록자료 대출 팝업

> "부록자료가 배가자료인지 확인이 필요합니다. 부록자료의 상태가 정리대상자료이면 [목록완성] 메뉴에서 부록자료를 열람인계처리합니다."

→ **함의**: 우리 SaaS는 부록자료 자동 인계 처리 후보.

### FAQ 3 — 별치기호 일괄 부여

> "[목록완성] 메뉴의 [일괄변경] 버튼을 통해 별치기호를 일괄로 부여할 수 있습니다."

→ **함의**: 우리 SaaS는 별치기호 일괄 부여 자동화 후보 (사서 시간 절감).

### FAQ 4 — 연속간행물 권호 예측

> "월간 연속간행물 권호 정보가 13월호, 14월호, 15월호로 예측됩니다. 권호 예측정보 '?월호'에 해당하는 단계증가 값을 12로 입력합니다."

→ **함의**: KOLAS III 권호 예측 알고리즘 단계증가 값 = 12 (월간) / 7 (주간) / 1 (일간). 우리 SaaS도 동일 적용.

### FAQ 5 — 수입구분 변경

> "[배가자료관리] 메뉴의 [코드변경]으로 소장자료의 수입구분을 변경할 수 있습니다. 코드변경창의 [수입구분] 항목에서 변경 가능합니다."

→ **함의**: 등록 후 수입구분 (구입↔기증) 잘못 입력 = 우리 SaaS 사전 검증 후보.
