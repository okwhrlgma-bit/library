## 1. 책단비 hwp 양식 4종 (자관 매일 작성)

| 양식 | 용도 | 행 패턴 |
|---|---|---|
| `책단비팀 공통 양식(만료).hwpx` (83KB) | 만료자료 띠지 | `<[내숲]만료[은평]><권>` × 65 |
| `책단비팀 공통 양식(반납).hwpx` (86KB) | 반납자료 띠지 | `<[내숲]반납[은평]><권>` × 65 |
| `책단비팀 공통 양식(제공).hwpx` (83KB) | 제공자료 띠지 | `<[내숲]제공[은평]><권>` × 65 |
| `책단비팀 공통 양식(제공) _ 첫장이름지운거.hwpx` (73KB) | 제공 익명 버전 | 동일 |
| `책단비팀 공통 양식(지하철).hwp` (59KB) | 지하철 도서관 전용 (HWP 5.0 OLE2) | 별도 양식 |

**띠지 1장 = 65권 분량** (도서관20·역30·여유15 = 65, 또는 ver.2 도서관20·역30 = 50권).

**양식 토큰 구조**:
```
<[자관명]><[상태]><[상대도서관]><권>
   ├ [내숲]       ← 자관 = ○○도서관
   ├ 만료/반납/제공 ← 상태
   ├ [은평]       ← 은평구 (또는 다른 자치구·지하철역)
   └ <권>          ← 책 권차 정보 입력 필드 (mail merge 자동)
```

## 2. 책단비 .xlsm 매크로 5년 历사 (1,328 파일)

| 연도 | 파일 수 | 패턴 |
|---|---:|---|
| 2018 | 22 | 첫 도입 (4분기 시작 추정) |
| 2019 | 358 | 본격 운영 = 거의 매일 |
| 2020 | 330 | 코로나 시기에도 유지 |
| 2021 | 349 | 회복 |
| 2022 | 327 | 안정 |

**.xlsm = Excel 매크로 매핑** — 자관이 5년간 매일 KOLAS 책두레 F12 엑셀 → 자체 매크로로 책단비 대장 자동 변환 → hwp 양식으로 출력.

**책단비띠지 ver.1 vs ver.2** (2019.12.10 동시 작성):
- ver.1: 도서관15 + 역15 = 30권 한 장 (102KB)
- ver.2: 도서관20 + 역30 = 50권 한 장 (113KB)

→ 자관이 자체적으로 ver 진화 시킴 = **사서가 도구 부족 인지 + 자체 해결 중** = 우리 SaaS 진입 정당성.

## 4. 우리 SaaS 신규 모듈 후보

### `chaekdanbi/auto_label_generator.py` (신규)

```python
def generate_chaekdanbi_label(
    kolas_f12_xlsx_path: Path,
    library_name: str = "내숲",      # 자관 → config.yaml
    target_libraries: list[str],      # ["은평", "마포", "서대문"] 또는 지하철역
    label_type: Literal["만료", "반납", "제공"],
    template_dir: Path = Path("data/chaekdanbi_templates/"),
    output_path: Path,
) -> Path:
    """
    KOLAS 책두레 F12 엑셀 → 책단비 hwp 띠지 자동 생성.
    
    1. xlsx 읽고 등록번호·서명·청구기호 추출
    2. 도서관별/역별 분류 (target_libraries 매처)
    3. hwpx 템플릿 mail merge
    4. 출력 hwp/hwpx 저장
    """
```

### 의존성 신규
- `python-hwpx>=0.5` (HWPX 파일 생성 — OOXML 기반)
- 또는 `olefile>=0.46` + 직접 HWP 5.0 OLE2 핸들링 (HWP 양식 전용)

### config.yaml 신규 섹션
```yaml
chaekdanbi:
  library_name: "내숲"               # 자관명 토큰
  default_target_libraries:           # 기본 상대 도서관·역
    - "은평"
    - "마포"
    - "서대문"
  ver: 2                              # 띠지 버전 (1·2)
  rows_per_label:
    library: 20                       # 도서관 행 수
    station: 30                       # 지하철역 행 수
  templates:
    expire: "data/chaekdanbi_templates/만료.hwpx"
    return: "data/chaekdanbi_templates/반납.hwpx"
    provide: "data/chaekdanbi_templates/제공.hwpx"
    subway: "data/chaekdanbi_templates/지하철.hwp"
```

## 5. 영업 메시지 (PO 직접 활용 — 정정)

### 은평구 11개 도서관 영업 (자관 양식 PILOT 후)

> "○○도서관이 5년 동안 매일 작성한 1,328 상호대차 대장을 분석했습니다.
> 사서 1명이 매일 26~31분 쓰던 책단비 띠지 작업을, 우리 SaaS는 6~7분으로 줄입니다 (시간 단축 77%).
> KOLAS 책두레 F12 엑셀 1번 다운받기만 하면, **자관 책단비 hwp 4 양식 (만료·반납·제공·지하철)** 이 자동 생성됩니다.
> 5년간 사서가 자체 매크로 ver.1·ver.2로 진화시켰던 작업을, 우리는 클릭 1번으로 끝냅니다."

### 다른 자치구·전국 도서관 영업 (표준 양식 기반 + 자관 양식 등록 가능)

> "각 자치구별 상호대차 명칭 (책두레·책나래·책바다·책이음 등 전국 표준 + 자치구 한정 자체 명칭)에 모두 호환됩니다.
> 자관이 자체 양식을 사용 중이면, **양식 업로드 후 우리 SaaS가 자동 mail merge** 합니다.
> 표준 양식 (KOLAS 책두레·책나래·책바다·책이음)은 기본 제공 — 별도 설정 불필요."
