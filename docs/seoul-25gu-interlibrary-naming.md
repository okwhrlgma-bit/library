## 4. 우리 SaaS 영업 우선순위 (Phase 2 자치구별)

### 🟢 1순위 — 자체 명칭 + 자체 양식 보유 자치구 (3건)

| 자치구 | 자체 명칭 | 영업 메시지 |
|---|---|---|
| 은평구 | 책단비 | 자관 PILOT 검증 데이터 직접 활용 (1,328 대장 5년 历사) |
| 양천구 | 책가방 | "양천 책가방 양식도 자관 PILOT 후 자동 mail merge" |
| 마포구 | 책마중 | "2018.7 정상운영 책마중도 우리 SaaS 호환" |

### 🟡 2순위 — 책이음 우산 자치구 (18건)

표준 양식 (책이음·책두레) 기반 자동화 = 우리 SaaS 기본 제공.
별도 자치구 양식 발견 시 업로드.

### 🟡 3순위 — 강남·강북 등 자체 명칭 X (4건)

일반 "상호대차" 명칭 사용 → 표준 양식 기반.

## 5. 우리 SaaS 양식 우선순위 (4단 fallback) 적용

```
사서가 책단비 작성 시도 → forms/resolver.py:
  1. 사용자 custom 양식? → 사용
  2. 표준 양식 (책이음·책두레·책나래·책바다)? → 표준 사용
  3. 자관 기본 양식 (data/forms/eunpyeong/chaekdanbi/*.hwpx)? → 자관 양식 (라이선스 동의 후)
  4. 모두 X → 빈 양식 + 사용자 설정 안내
```

자관 양식 사용 조건:
- 자관 (내숲) PILOT 동의서 후만 활성
- 다른 자치구 사서가 사용 시도 → "은평구 한정 양식 — 자치구 양식 업로드 필요" 안내
