# 책이음 운영자 교육 (2021.02.17) 정독 + KLMS 통합 시스템

> **출처**: 자관 D 드라이브 `0.책이음/` 5건 (총 21.9MB) + NLK 공식
> **분석일**: 2026-04-28
> **결론**: 책이음 = KOLASIII (자관) + KLMS (NLK 클라우드) **2-tier** 통합 시스템

---

## 0. 책이음 정체 (정정 + 강화)

| 항목 | 정보 |
|---|---|
| 정의 | **하나의 회원증으로 전국 참여도서관 이용** |
| 운영 | NLK + 문체부 (2015~2017 확장) |
| 참여 | 1,000+ 도서관 (서울 200+) |
| 시스템 | KOLASIII (자관) + **KLMS** (NLK 클라우드 통합) |
| 본인인증 | **마이핀 인증** 또는 **휴대폰 인증** (PII) |

---

## 1. KLMS = Korea Library Management System (NLK 클라우드)

**책이음 통합 통계·동기화 시스템** — KOLASIII와 별도.

| KLMS 메뉴 | 기능 |
|---|---|
| 도서관관리 → 동기화 수행 | 참여도서관·지역센터·NLK **3-tier 동기화** |
| 보정 | 수동 동기화 (특정 데이터 문제 시) |

**우리 SaaS 영역 X** — KLMS는 NLK 통합 시스템, 우리 SaaS가 직접 접근 X.

## 2. 책이음 회원 관리 규정

| 항목 | 값 |
|---|---|
| **통합대출 권수** | **20권** (문화가 있는 날 40권 조정) |
| 자관대출 권수 | 20권 내 자율 |
| **대출기간** | **15일** (대출일 포함) |
| 대출정지일 부여 기준 | 별도 |
| 자격 상실 기준 | 별도 |
| 재가입 허용 기준 | 별도 |

---

## 3. KOLASIII 책이음 신규 회원 가입 (5단계)

```
1. 메뉴: KOLASIII > 공통자료 > 대출/반납 > 이용자입력
2. 대출자번호의 번호구분 = 통합대출자 번호 선택
3. 책이음 회원 여부 = Y 선택
4. 본인인증 진행 (마이핀 또는 휴대폰)
5. 신규 책이음 회원으로 저장
```

→ KOLASIII 기본 버전 + 확장 버전 차이 존재.

---

## 4. 책이음 통계 (KOLASIII)

| 메뉴 | 통계 종류 |
|---|---|
| `KOLASIII > 공통자료 > 통계 > 책이음 통계` | 전체 등록회원 (전국) / 자관최초 등록회원 |
| `대출반납 > 책이음 통계` | 자관 책이음 이용 통계 |
| `대출반납 > 이용자의 책이음 서비스 대출이력 조회` | 책이음 이용자 대출 이력 |
| 통합센터대출이력조회 | 반납완료 자료 (대출중은 '통합대출현황'에서) |

---

## 5. 책이음 ↔ 책바다 ↔ 책나래 3-way 비교 (정밀)

| 시스템 | 본질 | 회원 인증 | 비용 | 우리 SaaS 영역 |
|---|---|---|---|---|
| **책이음** | 통합 회원증 (1 카드 → 다관) | 마이핀·휴대폰 본인인증 | 무료 | ❌ PII (회원 본인인증) |
| **책바다** | 국가상호대차 (책 자체 이동) | 책바다 회원가입 | 5,200원 (이용자 결제) | ❌ 결제 PII |
| **책나래** | 장애인 우편 (NLD) | 등록장애인 등 | 무료 (NLD 지원) | ❌ 민감정보 (장애 정보) |
| **책두레** | KOLAS III 모듈 (자관 간 상호대차) | 도서관 직원 | 무료 | ✅ F12 엑셀 자동 import |
| **책단비** | 자치구 자체 (은평구) | 자관 회원 | 무료 | ✅ 양식 자동 생성 |

**우리 SaaS 진입 영역 = 책두레 + 책단비** (PII 영역 없음).
**우리 영역 X = 책이음·책바다·책나래** (회원 PII·결제·민감정보).
