## 4. 강의 구성 (60분)

### 4-1. 이론 30분

#### 도입 (5분) — 사서 공감대
- 한국 도서관 사서 KORMARC 마크 작업 비효율 (권당 8분·1일 8~13시간)
- 1인 사서 학교도서관·자원봉사 사서 86%·작은도서관 1,800곳 도구 부재
- 자관 「내를건너서 숲으로 도서관」 사례 (xlsm 4,233개·5년 책단비 1,328건·6년 NPS)

#### 표준 정합 (10분) — KORMARC 2023.12
- KS X 6006-0:2023.12 (NLK 2차 개정) 9 자료유형
- M / A / O 적용 수준 분기 (필수·조건부·선택)
- 008 40자리 자료유형별 분기·245 지시기호·049 자관 청구기호
- 880 한자 페어·NLK 「서지데이터 로마자 표기 지침(2021)」
- 한국 KOLAS 실무 vs 학술 표준 차이 (RDA 336/337/338 미적용 다수)

#### 자동화 디테일 (10분) — kormarc-auto
- 외부 API 폴백 (NL Korea 1순위 → 알라딘 → 카카오)
- AI 결정 보조 (KDC 후보 3·NLSH 추천·사서 책임 영역 침범 X)
- 4단 검증 (어셔션·골든·시각 회귀·자관 .mrc 99.82%)
- KOLAS·알파스·KERIS DLS·KOLIS-NET·5 상호대차 import-export

#### 사서 책임 영역 보호 + PIPA (5분)
- 사서 결정 영역: KDC·청구기호·880 검수 (자동 X)
- PIPA 옵션 C: 회원 PII = 자관 알파스 위임·우리는 양식·통계만
- 도서관법 §21 납본·§22 폐기·§35-3 DSAR 자동

### 4-2. 실습 30분

#### 환경 셋업 (5분)
- 수강 사서 노트북 사전 준비: Python 3.12 설치 + GitHub repo clone
- `.env` 파일에 NL Korea·알라딘·카카오 키 채움 (사서 본인 발급)
- `kormarc-auto info` 1줄 → 환경 진단

#### Phase 0 MVP 실습 (10분)
- ISBN 13자리 입력 → `kormarc-auto isbn 9788912345678 --output-dir <KOLAS폴더>`
- 5초 안에 KORMARC .mrc 생성 → KOLAS 마크 반입 폴더 자동 인식
- 본인 도서관 신착 5권 일괄 처리 (`kormarc-auto batch sample.txt`)

#### 049 prefix 자동 발견 (5분)
- `kormarc-auto prefix-discover "D:\<자관>\수서"` 1줄
- 자관 049 ▾l prefix 분포 + config.yaml snippet 즉시 출력
- 자관별 정합 (자관 = EQ·CQ·WQ → 99.82% 정합 사례)

#### 책단비·납본·폐기 (10분)
- 책단비 hwp 4 양식 자동 (`kormarc-auto interlibrary --type chaekdanbi`)
- 납본 별지 제3호서식 PDF (`kormarc-auto deposit form`)
- 분기 제적심의 결재서식 + 폐기목록 엑셀 (`kormarc-auto dispose`)
