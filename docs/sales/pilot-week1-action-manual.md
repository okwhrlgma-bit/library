### 0.1. 사서 첫 만남 임팩트 — sanity-check 보고서 (★ NEW v0.4.38+)

```powershell
kormarc-auto sanity-check "D:\내를건너서 숲으로 도서관\수서" --json reports/sanity_2026-05-week1.json
```

출력 예시 (조기흠 사서 화면 첫 인상):
```
==================================================
자관 .mrc 진단 보고서

디렉토리   : D:\내를건너서 숲으로 도서관\수서
파일 수    : 174건
레코드 수  : 3,383건
정합 레코드: 3,377건
위반 레코드: 6건
정합률     : 99.82%
--------------------------------------------------
049 prefix 분포 (상위 5):
  EQ:  2553건 ( 75.5%) [권장]
  CQ:   773건 ( 22.8%) [권장]
  WQ:    57건 (  1.7%) [권장]
==================================================
```

**PO 발화 (그대로 사용)**:

## 1. 시연 30분 정확한 시나리오

### 1-1. 인사 + 4주 PILOT 구조 안내 (5분, 18:00~18:05)

**PO 발화 (그대로 사용)**:

> "안녕하세요 조기흠 사서님. 오늘부터 4주 동안 제가 만든 KORMARC 자동화 시스템을 사서님께 보여드리고 의견을 받으려고 합니다. 1주차는 오늘 30분, 이번주 신착 5권을 함께 처리해보고요. 2주차는 책단비 hwp 자동, 3주차는 수서 박지수 사서님과, 4주차는 종합 4명과 진행할 예정입니다."

> "오늘 시연은 ISBN 1번 입력 → KORMARC .mrc 5초 생성 → 자관 KOLAS에 실제로 반입까지 보여드릴게요. 마지막 5분은 이게 사서님 본인 예산으로 결제하실 만한지 직접 질문드릴 거고, 정직하게 NO도 좋습니다."

### 1-2. Phase 0 MVP 데모 — ISBN 1권 처리 (5분, 18:05~18:10)

**진행**:
1. PO PowerShell 열고 (사서가 화면 함께 봄)
2. 신착 1권 ISBN 입력:
```powershell
kormarc-auto isbn 9788912345678 --output-dir "D:\KOLAS\Import"
```
3. 5초 안에 출력:
```
✓ 9788912345678.mrc 생성 (1.2 KB)
출처: NL Korea (신뢰도 0.95)
필드 12개, 한자 880 자동 페어 1건
```
4. 자관 KOLAS 마크 반입 화면으로 전환 → 자동 인식 확인 → 사서가 화면으로 검수.

**조기흠 사서 즉시 코멘트 받기** (PO 카카오 녹음 또는 메모):
- "수동으로 하면 몇 분 걸리시나요?" → 답변 기록 (일반적으로 8~15분).
- "표제·저자·KDC·880 한자 자동 처리 정확도는요?" → 답변 기록.

### 1-3. 자관 신착 5권 실시간 변환 (10분, 18:10~18:20)

**진행**:
1. PO가 5권 ISBN을 sample_isbns.txt에 줄당 1개로 저장.
2. 일괄 처리:
```powershell
kormarc-auto batch sample_isbns.txt --output-dir "D:\KOLAS\Import"
```
3. 5권 모두 30초 안에 처리.
4. KOLAS 반입 폴더 확인 → 5건 자동 인식 → 사서 검수.

**측정**:
- 시간: ★ PO 측정 (목표 < 5초/권)
- 정합: 5건 모두 KOLAS 반입 성공 ☐
- 한자 880 자동: ___ /5 건
- KDC 자동: ___ /5 건 (사서가 검수 후 OK 또는 수정)

**조기흠 사서 코멘트** (자유 발화 받기):
- "일반적으로 5권 처리 시간 vs 우리 시스템 시간 차이는?"
- "사서가 검수해야 할 부분은?" (사서 책임 영역 강조)
- "본인 매크로 (xlsm 200KB+)와 비교하면?"

### 1-4. Q1 결제 의향 직접 질문 (5분, 18:20~18:25)

**PO 발화** (그대로):

> "마지막 5분은 솔직한 질문 드릴게요. 사서님 같은 매크로 사서를 위한 플랜이 월 5만원 (월 1,000건 처리)인데, 본인 예산으로 결제하실 의향이 있으신가요? 도서관 예산이 아닌 사서님 본인 예산 기준으로요."

**Q1 답변 4단 기록**:

| 답변 | 점수 | 기록 |
|---|---|---|
| HIGH | 80~100 | "지금 당장 결제 가능" |
| MID | 50~79 | "검토 후 결제 검토" |
| LOW | 20~49 | "있으면 좋은데 본인 예산은 어려움" |
| 0 | 0~19 | "결제 의향 없음" |

**보강 질문** (Q1 답변에 따라):
- HIGH/MID: "5월 中 정식 결제 가능하신가요? 첫 50건 무료 체험 + 다음달 결제."
- LOW: "어떤 조건이면 결제 가능하실까요? (가격·기능·기간 등)"
- 0: "왜 결제 의향이 없으신지 솔직하게 알려주세요. (가격·도구 X·신뢰 X 등)"

### 1-5. NPS 1차 측정 (3분, 18:25~18:28)

**PO 질문**:

> "오늘 시연한 시스템을 다른 사서에게 추천할 의향이 0~10점 중 몇 점이신가요?"

**기록**:

**NPS 분류**:

- 7~8: 중립 (Passive)
- 0~6: 반대자 (Detractor)
