## 1. 우리 사업의 정부 정당성 — 4가지 정부 권고 인용

### 1.1 정부가 인정한 사업 방향 (보고서 §3 결론·§4.3)

> "AI 기술을 활용하여 주제색인 과정 및 서지레코드 생성 프로세스의 효율화 필요" (p.182)
>
> "AI 기술을 활용한 주제색인 품질 제고" (p.182)

**의미**: 국회도서관 자체가 AI 기반 KORMARC 자동화를 **정책 방향으로 명시**. 우리 SaaS는 정부 권고 직접 구현.

### 1.2 사서 검수 (HITL) 의무화 정부 권고 (p.184·185)

> "수작업으로 할당된 주제와 알고리즘이 제안한 색인 간의 유사도에 대한 검증이 필요함" (p.184)
>
> "위 두 가지 방식으로 할당된 색인에 대한 전문가의 평가 과정도 필수적임" (p.184)

**의미**: 우리 헌법 §1 (의심·검증·정제 3원칙) + ADR 0009 정신 = 정부 공식 권고와 정확히 일치. "AI 단독 자동화"가 아닌 "AI 보조 + 사서 검수"는 정부 권고 디자인 패턴.

### 1.3 ChatGPT 단독 KORMARC 생성 한계 — 보고서 실험 결과 (p.156, 표 Ⅲ-26·Ⅲ-27·Ⅲ-28)

**ChatGPT API로 KORMARC 30건 도서명 → 주제명 자동 부여 실험**:

| 평가 항목 | 결과 |
|---|---:|
| KORMARC 형식 적합성 | **100%** (모든 주제명이 답변 형식에 적합) |
| 국회도서관 용어관계사전 출현 | **25.8%** (30건 중 8건만 식별 — 74.2%가 비우선어 또는 미존재) |
| 기존 주제명과 의미적 완전일치 | **42.1%** (19개 비교) |
| 부분일치 | **36.8%** |
| 완전불일치 | **21.1%** |

> "ChatGPT를 활용하여 목록 레코드 생성 시 이에 대한 한계점과 편향 가능성을 염두에 두어야 함" (p.125, Brzustowicz 2023 비판 3건 인용)

**우리 사업 정합**:
- 우리 4단 폴백 (NLK SeoJi → 정보나루 → Aladin → Claude Vision) = ChatGPT 단독 의존 X
- **권위 통제 vocab 연계 (NLSH·KDC) = 우리 차별점** — ChatGPT 단독 25.8% vs 우리 통제어휘집 연동으로 ↑
- 사서 검수 UI = 21.1% 완전불일치 보정

### 1.4 자동색인 도구 개발 정부 권고 (p.182·185)

> "AI 기술을 활용한 주제색인 품질 제고가 가능한 주제색인 도구 개발이 필요함" (p.6 요약문)
>
> "다국어 지원·다양한 통제어휘집 수용·다양한 주제색인 알고리즘 지원·다양한 인터페이스(명령어, 웹, API 등)·오픈소스 소프트웨어로 제공" (p.185)

**의미**: 정부가 권고하는 자동색인 도구 5가지 요건 모두 우리 SaaS 정합:
- 다국어 (한국어 + 한자 880 페어)
- 통제어휘집 (NLSH·KDC 6판)
- 다양한 인터페이스 (CLI·Streamlit·FastAPI)
- ADR 0008 Apache 2.0 코어 (오픈소스)

## 2. Annif 도입 검증 — 정부 권고 모델 (p.127~130)

> "Annif: 핀란드 국립도서관 개발, 다양한 알고리즘·인터페이스·오픈소스 소프트웨어" (p.127)
>
> "Annif의 9개 알고리즘 중 7개가 F1@5 점수에서 현재 솔루션을 능가" (p.129)
>
> "도서 유통업체 'Kirjavälitys Oy': 출판사로부터 도서 정보 → 도서관·서점·핀란드 국가서지 Fennica·종합목록 Melinda 메타데이터 생성" (p.130)

**우리 도입 시점**:
- ADR 0014 후보 (KDC 분류 단순화 — ISBN 부가기호 broad class 95%+) 적용 후 → AI 폴백 단계에서 **Annif Apache 2.0 채택**
- BERT 기반 사전학습 모델 (bert-base) — 우리 prompt caching·diskcache 정합
- 비용: $0 (오픈소스, GPU·CPU 모두 가능)

**정부 권고 명시 = 우리 도입 시 영업 자료에 인용 가능**:
> "정부 (국회도서관·공주대) 2024년 보고서가 권고한 Annif 알고리즘 (F1@5에서 핀란드 국립도서관 현 솔루션 능가)을 우리는 KDC 자동 분류 폴백에 도입"

## 3. BIBFRAME 4단계 — 우리 진입 시기 정부 검증 (p.187, 그림 Ⅳ-16)

| 단계 | 기간 | 작업 |
|---|---|---|
| 1단계 | **2025-2028** | 서지구조 변화 기반 (SKOS·MADS·어휘 인코딩스킴·교육) |
| 2단계 | 2028-2030 | 홈페이지·입력기 (오픈소스 기반)·시범 적용 |
| 3단계 | 2030-2032 | 변환기·BIBFRAME 입력기·MARC+BIBFRAME 듀얼 |
| 4단계 | 2032+ | 본격 BIBFRAME 시스템 전환 |

**의미**:
- **2028년까지 KORMARC가 주류** = 우리 3~5년 윈도우 (PO 6개 검증 보고서 ⑤ 정합)
- 2028년 이후 BIBFRAME 듀얼 = 우리 BIBFRAME 변환 모듈 추가 시점 (ADR 0009 §33 미국 활성화 트리거와 유사 구조)
- 사서 교육 의무 (p.181) = 우리 사서 친화 UI + 5분 시연 체크리스트 정합

## 4. 국회도서관 자체 한계 인정 (p.169·170) — 우리 진입 기회

> "도서관에서는 양질의 데이터를 구축하고 있었으나 구축한 포맷이 MARC 형태이므로 외부에서 접근하여 데이터를 이용하기가 어려움" (p.169)
>
> "국회도서관의 데이터를 공개하여 이를 외부에서 활용 가능하도록 함으로써 도서관 Ecosystem 구축" (p.170)

**의미**: 정부도 KORMARC의 외부 접근 어려움을 공식 인정. 우리 API 우선 + 모바일 (폰 카메라) + 사서 친화 UI = 정부 권고 Ecosystem의 민간 구현.

### 국회도서관 데이터 규모 (p.163·164·187, 영업 인용 가능)

| 항목 | 건수 |
|---|---:|
| 용어관계사전 (시소러스) 주제어 | **129,339건** |
| 전거데이터 (개인·단체) | **1,779,238건** |
| 주제분야 분포 (167,089건) | 경영/경제 14.8% · 사회과학 14.4% · 법 13.8% · 의학/생명과학 9.5% · 행정 8.4% · 정치 7.9% · 기타 31.2% |

**우리 사업 활용**:
- NLSH 통제어휘집 연동 시 우리 정확도 ChatGPT 단독 25.8% → **80%+ 향상 시뮬 근거**
- 영업 메일에 "국회도서관 시소러스 12.9만 + 전거 178만 = 한국 도서관 최대 권위 통제 어휘집과 직접 연동" 인용

## 5. 정부 권고 자동색인 워크플로우 (p.185~186) — 우리 헌법 §1 정합

```
[자료 입력]
    ↓
[자동색인 도구가 색인 워크플로우 통합]
    ↓
[수작업 색인 ↔ 알고리즘 색인 유사도 검증]   ← 우리 confidence 점수
    ↓
[전문가(사서) 평가]                       ← 우리 사서 검수 UI
    ↓
[기존 서지레코드 데이터 요소 보강 (요약·목차)]
    ↓
[생성형 AI로 주제어 색인 보강]            ← 우리 4단 폴백
    ↓
[색인 작업 포함된 목록 워크플로우 본격 도입]
```

**의미**: 우리 `aggregator.py` (4단 폴백) + `confidence` 점수 + 사서 UI 검수 = **정부 보고서 권고 워크플로우 정확 구현**.

## 6. 영업·정부지원사업 인용 핵심 문구

### 사업계획서·예비창업패키지

> "본 사업은 국회도서관·공주대학교 산학협력단 2024년 12월 발간 「인공지능 시대 국회도서관 지식정보 구조화 방안 연구」(이미화 책임연구원, 200쪽)에서 권고한 자동색인 도구 개발 방향을 정확히 구현한다. 보고서는 'AI 기술을 활용한 주제색인 품질 제고가 가능한 주제색인 도구 개발이 필요함'(p.6)·'다국어·통제어휘집·다양한 인터페이스·오픈소스 5가지 요건'(p.185)을 명시했으며, 본 SaaS는 5가지 요건을 모두 충족한다."

### KOLAS 영업 메일

> "국회도서관 자체 실험(p.156, 30건 KORMARC 도서명 ChatGPT 자동 색인)에서 단독 자동화는 의미적 완전일치 42.1%·국립중앙도서관 용어관계사전 출현 25.8%로 한계가 명확합니다. 본 SaaS는 4단 폴백(국립중앙도서관 SeoJi → 정보나루 → 알라딘 → AI Vision)과 사서 검수 UI를 결합하여 정부 권고 워크플로우(p.185 '수작업 + 알고리즘 유사도 검증 + 전문가 평가')를 정확히 구현합니다."

### 컨소시엄·작은도서관협회 피치

> "BIBFRAME 4단계 정부 계획(p.187, 그림 Ⅳ-16)에 따르면 2028년까지 KORMARC가 주류이며, 작은도서관·학교도서관에 우선 적용 가능한 자동화 도구는 정부 정책상 향후 4년간 시장 적기입니다."

### 학술·언론 인터뷰

> "국회도서관 보고서는 '양질의 도서관 데이터가 MARC 형태라 외부 접근이 어렵다'(p.169)고 지적하며 '도서관 Ecosystem 구축'(p.170)을 권고했습니다. 본 SaaS는 정부 권고 Ecosystem의 민간 구현체로서, 작은도서관·학교도서관 사서가 모바일에서 책 사진 1장으로 KORMARC를 생성하는 첫 한국 SaaS입니다."
