## A.1 한국 장애인 도서관 시장 (TAM)

### 법적 근거
- **장애인복지법** = 장애인 정보 접근 권리
- **도서관법** = 모든 도서관 = 장애인 서비스 의무
- **장애인차별금지법** = 디지털 정보 동등 접근
- **저작권법** §33의2 = 장애인용 변환 (예외)

### 시장 규모
- **국립장애인도서관 (KNLB)** = 1관 (서울)
- **점자도서관** = 약 30관 (전국)
- **공공도서관 장애인 자료실** = 800+관
- **대학도서관 장애학생 지원** = 200+관
- **시각장애인 등록자** = 25만+
- **청각장애인 등록자** = 40만+
- **TAM** = 1,000+ 도서관·1만+ 장애인 사서·30만+ 이용자

### 무시 비용
- 진입 X = 1,000관 시장 손실
- KWCAG 미달 = 장애인차별금지법 위반 가능성
- CSR 메시지 X = ETH1·G1 약화

## A.3 장애인 보조기기 호환 (kormarc-auto)

### 1. 점자정보단말기 (Braille Display)
- **기기**: HumanWare BrailleNote·Vario·Focus 등
- **호환**: 화면 낭독기 NVDA·JAWS 표준 출력 → 점자 단말기 자동
- **보완**: KWCAG 2.2 AA 정합 (E2 검증) = 자동 호환

### 2. 음성 합성 (TTS·Screen Reader)
- **기기**: NVDA (무료)·JAWS·VoiceOver (Mac·iOS)·TalkBack (Android)
- **호환**: ARIA labels·시맨틱 HTML
- **보완**: 모든 화면 = 화면 낭독기 호환 검증
  - 단, **음성 출력 = 사서 환경 정숙 = 시각장애 사서 본인만** (Part 57 정합)

### 3. 확대 독서기 (Magnifier·CCTV)
- **기기**: 확대 독서기·전자 돋보기
- **호환**: 큰 폰트 (16px+)·고대비 모드·zoom 가능
- **보완**: 글씨 크기 조정 (사용자 토글)

### 4. 음성 인식 (Voice Input)
- **기기**: Dragon·Apple Dictation·Google Voice
- **호환**: 표준 input 필드
- **보완**: 모든 input = 음성 인식 호환

### 5. 화상 키보드·스위치
- **기기**: 신체장애 = 화상 키보드·1버튼 스위치
- **호환**: Tab·Enter 표준 = 자동 호환
- **보완**: 모든 기능 = 키보드 only (Part 57 적용 ✅)

## B.2 도서관 홈페이지 = 시장 진입 채널

### 사서 일과 → 홈페이지 운영 = 핵심

→ **kormarc-auto = 사서 일과의 도구 = 도서관 홈페이지 자동화 엔진** 포지셔닝

- "도서관 홈페이지 신간·추천 자동 갱신 = 사서 시간 5x 절약"
