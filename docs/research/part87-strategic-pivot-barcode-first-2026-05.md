## 3. KORMARC 표준 정정 사항

### 3.1 자료유형 = 8 (NOT 9)

KORMARC 통합서지용 표준 8 카테고리:

| 코드 | 명칭 |
|---|---|
| BK | 도서 |
| CR | 계속자료 |
| CF | 전자자료 |
| MP | 지도자료 |
| MU | 녹음자료 |
| VM | 시청각자료 |
| RB | 고서 |
| MX | 복합자료 |

**현재 코드 상태**: `src/kormarc_auto/kormarc/material_type.py`에 17 granular subtype 정의 (book_single·book_multi·thesis·braille·rare_book 등).
- 17 subtype = KORMARC 8 카테고리에 매핑되는 세분화 (정합 ✅)
- "9 자료유형" 마케팅 표현 = doc 통계 (libsta.go.kr) 분류와 혼동 → README/CLAUDE.md 정정 필요

### 3.2 카탈로깅 시간 최다 필드
245(표제) · 700(부출표목) · **650(주제명)** · 505(목차) · 520(요약) · 056(KDC).

## 4. API 백본 폭포수 (M3)

### 4.1 권장 폭포수 아키텍처
```
Scan(EAN-13)
  ↓
SEOJI API (1차·무료·공공누리) → 12 필드
  ↓ 미스
data4library API (KDC 056 보강·무료)
  ↓ 미스
NL Open API (레거시 도서 폴백)
  ↓ 미스
알라딘 TTBKey (도서관 자체 키 위임만·505/520 보강)
  ↓ 미스
Naver/Kakao (이중화 안전망)
  ↓ 모두 미스
사진 OCR + LLM 폴백
```

### 4.2 부가기호 5자리 = KDC 무비용 추출
EAN-13 add-on 5자리에서:
- 1행 = 독자대상
- 2행 = 발행형태
- 3-4행 = KDC 대분류 + 세분류
- 5행 = 예비

예: `03810` = 교양 · 단행본 · 문학 · 한국문학.

→ SEOJI KDC 공백 (2020-12-31 이후 미제공) 즉시 해소. **0 API 호출·100% 정확**.

### 4.3 알라딘 위임 모델
**약관**: "도서 판매·도서 정보 기반 영리 서비스 이용 불가"

**현재 사용 패턴**: 위반 위험 50%+

**해결**: `ALADIN_TTBKEY_PER_TENANT` 환경변수 → 도서관 자체 계정 키 위임 호출만.

→ 우리 SaaS = 도구 제공자 / 도서관 = 알라딘 키 보유자 = 약관 위반 X.
