## 0. 사서 디자인 핵심 통찰 (T5 UX Researcher 시각)

### 사서가 일반 SaaS 사용자와 다른 5 차이

1. **반복 작업 = 일주일 100~500권 = 피로 회피 핵심**
2. **정확 강박 = 1글자 오류 = KOLAS 반입 실패 = 절대 X**
3. **공용 환경 = 카운터·이용자·동료·관장 시야**
4. **노후 PC = Windows 10·저사양·낡은 모니터**
5. **표준 강박 = KORMARC·KCR4·NLK 정확 정합**

→ **사서 디자인 = 일반 B2B SaaS 디자인과 5 차원 다름**

## 1. 사서가 원하는 디자인 8 핵심 (사서 인터뷰·페르소나 시뮬 종합)

### A. "내 일을 도와주는 도구" (Tool, not Product) ★★★★★
- **원함**: 사서 워크플로 = 절대 = 도구는 보조
- **거부**: 화려한 UI·복잡 기능·자랑·과시
- **디자인 원칙**:
  - 첫 화면 = 즉시 일 시작 가능 (5초 룰)
  - 메인 액션 = 단 1개 (ISBN 입력·일괄 처리)
  - 보조 기능 = 우측·하단 격리
  - "Look how cool" 디자인 X = "Just works" 디자인 ★

### B. "노후 PC도 빠른" (Performance > Aesthetics) ★★★★★
- **원함**: 페이지 로딩 ≤2초·반응 즉시
- **거부**: 무거운 애니메이션·hover 효과·박스 그림자
- **디자인 원칙**:
  - lite mode 토글 (Part 57 적용 ✅)
  - 애니메이션 = 의미 있을 때만 (성공·로딩)
  - SVG > PNG (확장·용량)
  - 첫 화면 LCP ≤1초

### C. "키보드만으로 모두" (Keyboard-First) ★★★★★
- **원함**: 마우스 X·키보드 only 100% 작업
- **거부**: 모달·드롭다운 필수·hover 의존
- **디자인 원칙**:
  - Tab·Shift+Tab 완벽 작동
  - 단축키 (Ctrl+S·Ctrl+Enter·Esc) 노출
  - Enter = 다음 단계 (form submit)
  - 모든 기능 = 키보드 가능 (KWCAG 2.2 정합)

### D. "검증·정확 가시화" (Trust through Transparency) ★★★★★
- **원함**: AI 결과 = 출처·신뢰도 표시 = 사서 검증 가능
- **거부**: 블랙박스·"믿어주세요" 메시지
- **디자인 원칙**:
  - confidence 표시 (90%·80% 시각화)
  - source_map 노출 (NLK·알라딘·카카오)
  - "AI 추천이지만 검증 필요" 명시
  - 자관 prefix·자동 추천 = 사서 확인 단계

### E. "사서 손에 익숙한 패턴" (Familiar Mental Model) ★★★★
- **원함**: KOLAS·알파스·DLS 기존 UI 패턴 익숙함
- **거부**: 완전 새로운 UI·학습 곡선
- **디자인 원칙**:
  - KORMARC mrk 형식 = 사서 익숙 (직접 노출)
  - 008·245·880 = 표준 라벨 (변경 X)
  - 자관 prefix·청구기호 = 자관 양식 그대로
  - 표·목록 = Excel 패턴 (사서 익숙)

### F. "이용자 옆에서도 안전" (Privacy-Aware) ★★★★
- **원함**: 카운터 옆 이용자 시야 보호
- **거부**: 민감 정보 화면 노출
- **디자인 원칙**:
  - privacy mask 토글 (Part 57 적용 ✅)
  - 개인정보·등록번호 마스킹
  - 화면 잠금 단축키 (Ctrl+L)
  - 자동 잠금 15분

### G. "정숙한 피드백" (Quiet Feedback) ★★★★
- **원함**: 시각만·소리 X (도서관 정숙)
- **거부**: 알림음·beep·음성 안내
- **디자인 원칙**:
  - 색·아이콘·텍스트 3중 (Part 57 적용 ✅)
  - 토스트 = 우상단 슬라이드 (소음 X)
  - 진동·소리 X
  - 시각 강조 (색·아이콘) 충분

### H. "사서 호칭·따뜻함" (Respectful Tone) ★★★★
- **원함**: "선생님" 호칭·전문성 인정·격려
- **거부**: 반말·기술자 톤·차가움
- **디자인 원칙**:
  - 메시지 = "선생님" 일관 (Part 49 ✅)
  - 70+ 사서 어휘 (수서·정리·배가)
  - 빈 상태 = 격려 ("처음이신가요? 함께 시작해요")
  - 에러 = 친절 ("괜찮아요, 다시 시도해보세요")

## 2. 사서가 필요한 디자인 컴포넌트 12 (T12 Design System)

### 입력 (Input)
1. **ISBN 입력 칸** = 13자리 자동 검증·실시간 피드백
2. **검색 박스** = 키워드·제목·저자·자관 전체 검색
3. **일괄 텍스트** = 1만 권 ISBN 텍스트 박스
4. **파일 업로드** = CSV·xlsm·이미지 (3종 인풋)
5. **자관 prefix 입력** = 자동 완성·추천

### 결과 (Result)
6. **KORMARC 결과 카드** = 008·245·880·049 라벨 + 편집 가능
7. **MRK 형식 토글** = 사서 익숙 mrk 직접 노출
8. **시간 측정 대시보드** = "8분 → 1분" 가시화

### 작업 흐름 (Workflow)
9. **단계 진행 표시** = "1/5 입력 → 2/5 결과 → 3/5 검토..."
10. **draft autosave 알림** = "30초마다 자동 저장됨"

### 시스템·환경
11. **lite mode 토글** = 노후 PC 정합
12. **privacy mask 토글** = 카운터 환경 정합

## 3. 사서가 필요한 정보 구조 (T13 IA)

### 사서 일과 정합 메뉴 트리

```
[홈]
├── 등록·정리 ★ 메인 일과 (수서·정리)
│   ├── ISBN 단건
│   ├── ISBN 일괄
│   ├── 책 표지 사진
│   └── 검색 (키워드·제목)
│
├── 자관 양식 (배가·납본)
│   ├── 자관 prefix 설정
│   ├── 청구기호 자동 생성
│   ├── 라벨 PDF 출력
│   └── 납본 양식 (NLK·교육청)
│
├── 일괄·마이그
│   ├── KOLAS·KLMS·KORIBLE export
│   ├── DLS export (학교)
│   ├── xlsm·alpas 마이그
│   └── 백업·복원
│
├── 통계·보고
│   ├── 사서 개인 통계 (시간 절감)
│   ├── 자관 종합 (관장용)
│   ├── 분기 보고서 (자치구)
│   └── 사서 평가 자료 (PR)
│
└── 설정·도움
    ├── 자관 설정 (prefix·청구기호 룰)
    ├── 사용자 모드 (사서/자원봉사)
    ├── 단축키 안내
    ├── 가이드 50+
    └── 1:1 문의
```

→ **사서 일과 = 5 메뉴 = 직관적·익숙·예측 가능**

## 4. 사서가 원하는 색·타이포·레이아웃 (T11 Product Designer)

### 색 (Color Palette)

| 색 | HEX | 용도 |
|----|----|----|
| 네이비 (Primary) | `#2C5282` | 도서관 톤·헤더·CTA |
| 살구 (Accent) | `#ED8936` | 강조·다운로드·중요 액션 |
| 아이보리 (BG) | `#FAFAF7` | 배경·따뜻함 |
| 짙은 회색 (Text) | `#1A202C` | 본문·읽기 편함 |
| 회색 (Muted) | `#718096` | 보조 텍스트 |
| 초록 (Success) | `#38A169` | 성공·완료 |
| 황금 (Warning) | `#D69E2E` | 경고·주의 |
| 빨강 (Error) | `#C53030` | 오류·실패 |

→ 차분한 도서관 톤·사서 친화·하루 종일 봐도 피로 X

### 타이포

- **본문**: 16px (50대 사서 대응) — Pretendard 또는 Noto Sans KR
- **H1**: 28px (5초 룰 = 명확)
- **H2**: 22px
- **H3**: 18px
- **caption**: 15px (Part 57 = 14→15 상향)
- **줄 간격**: 1.6 (가독성)
- **letter-spacing**: -0.02em (한글 정합)

### 레이아웃

- **컨테이너 max-width**: 760px (모바일·PC 양립)
- **버튼 min-height**: 48px (모바일 터치 안전)
- **카드 padding**: 16px
- **모서리 radius**: 8~12px (둥근·따뜻함)
- **그림자**: 약함 (lite mode 시 제거)
