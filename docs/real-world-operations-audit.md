## ⚠ 단기 (1주~1개월) 조치 권고 — Top 10

1. **HTTPS 정식 인증서** — Cloudflare Tunnel은 자동 TLS이지만 무료 도메인이 바뀌면 사서 북마크 무효화. 영구 도메인(`api.kormarc.kr` 등) 구매 + cloudflared `tunnel route dns` 고정.
2. **이메일 인증 도입** — 현재 `signup.py`는 이메일 형식만 검사 (`_EMAIL_RE`). 가짜 이메일로 `secrets.token_urlsafe(24)` 키 무한 생성 가능. 매직 링크 또는 6자리 OTP 추가.
3. **사용량 DB JSON → SQLite 마이그레이션** — 동시 사서 100명 시 `usage.json` 전체 read/write 락 충돌. SQLite WAL 모드로 무중단 전환.
4. **외부 API 다중 키 풀(pool)** — NL Korea 일일 한도 30,000건, 알라딘 5,000건, 카카오 30,000건. 사서 50명 × 평균 20건 = 1,000건/일이지만 **신학기 폭주 시(3월·9월) 한순간 초과 가능**.
5. **결제 플로우 — 토스페이먼츠 또는 포트원 가맹** — 현재는 "계좌이체/카카오페이 PO 직접 입금" → 학교/공공 결제팀 거부. PG 도입 + 자동 세금계산서 (사업자 등록 후 1주).
6. **약관 가입 동의 화면** — Streamlit UI 첫 화면에 체크박스 2개 (이용약관·개인정보처리방침) + 미체크 시 가입 차단. 현재는 약관 문서가 docs/에만 존재.
7. **카카오 채널 CS 개설** — 사서 90%는 이메일보다 카톡 선호 (`business-checklist.md`). FAQ 자동응답 + 영업일 24시간 SLA 명시.
8. **도서 표지 이미지 직접 저장 금지 재확인** — `regulatory-landscape.md`는 "URL만 저장" 명시했지만 Vision OCR이 사진을 base64로 캐시할 때 출판사 표지가 디스크에 남는지 점검 (`.cache/`).
9. **로그 보관 기간 정책** — 개인정보보호법 §21 (불필요해진 정보 즉시 파기). 현재 `logs/usage.jsonl`·`logs/signups.jsonl` 무한 누적. 90일 자동 회전 + 익명화 시점 명시.
10. **사서 이탈 시 데이터 다운로드/삭제 권리 (개인정보 자기결정권)** — 베타 약관에는 "30일 내 삭제" 있으나 자동 절차 없음. `/account/export`·`/account/delete` 엔드포인트 신설.

## A~N 항목별 상세 정리

### A. 개인정보 보호 (개인정보보호법)

**현 수집 항목**: 이메일·도서관명(선택)·API 키 SHA256 해시·사용 로그 (ISBN·시각·성공여부). 키는 익명화. 기본은 양호.

**리스크**:
- 가입 동의 화면 부재 (Streamlit UI에 체크박스 미통합)
- 한국 도서관 API 응답에 **이용자 식별 정보 누출 위험 0**: 우리는 ISBN·서지만 받음 (이용자 정보는 KOLAS/DLS 내부에만 존재) → 안전.
- 그러나 **Vision으로 책 사진 받을 때 이용자 손/얼굴 우연 촬영 가능** → 자동 모자이크 또는 안내문 필요.
- 2025년 개정안: **개인정보전송요구권** (사용자 자기 데이터 다운로드 권리) — 우리는 베타 약관에 명시는 했으나 자동 엔드포인트 없음.

**ISMS-P**: 의무 대상 한참 미달 (매출 100억 미만). 그러나 공공도서관 조달 가산점.

### B. 저작권·납본 (도서관법·저작권법)

- **KORMARC 데이터 자체**: 사실의 정보 → 저작권 없음.
- **알라딘 약관 (HIGH)**: "영리 서비스 이용 불가" 조항 발견. 우리 유료 SaaS는 **회색지대**. 별도 상업 계약 필요 가능. → 키 회수 시 4순위 폴백 손실. **PO가 알라딘 OpenAPI 담당자에게 서면 문의 권장**.
- **국립중앙도서관**: 공공누리 1유형 + CC0 1.0. **상업 이용 가능, 출처 표시만**.
- **카카오 책 검색**: 일 30,000건 무료, 비상업 제한 명시 없음. 그러나 약관 변경 가능.
- **도서 표지**: URL 노출만 (썸네일 임시 표시). 다운로드·재배포 X. 사진 OCR 시 base64 캐시가 디스크에 남는지 재확인.
- **납본 (도서관법 §20)**: 우리는 **납본 의무 주체가 아님** (출판사 의무). 그러나 사서 도구로 "납본 안내" 기능 추가 시 잘못된 안내 = 출판사 과태료 → 우리 책임으로 옮겨올 수 있음. **약관에 "법령 정보는 참고용, 실제 의무 이행은 사용자" 명시 필수**.

### C. 데이터 무결성·백업

- 자관 `.mrc` 누적 인덱스: `inventory/library_db.py` 기반. **OneDrive 자동 동기화 외 별도 백업 X**.
- 사용량 로그(`logs/usage.jsonl`)·결제 증빙: PC 디스크 사망 시 분쟁 시 증거 부재.
- **권고**: 매주 1회 `scripts/backup_logs.py` cron + S3·Google Drive 이중화. 정전 시 SQLite WAL 자동 복구.
- 디스크 풀: `.cache/kormarc-auto` 자동 정리 정책 (30일 TTL은 있지만 용량 한도 없음).

### D. 오류 회복

- 외부 API (NL Korea·알라딘) 폴백 5단계 잘 갖춰짐 (`api/aggregator.py`).
- **우리 서버 다운 시**: 사서가 KORMARC 작업 즉시 중단. 자동 알림 채널 없음 → UptimeRobot + 카카오 비즈 메시지.
- 도서관 PC 약한 네트워크: timeout=10s는 합리적이나, 모바일 4G에서 Vision은 사진 업로드 5~10초. **로컬 폴백 (이미지 압축 후 큐)** 권장.

### E. 성능·스케일

- **현 한계**: JSON 파일 락 — 동시 사서 ~10명까지 안전. 100명 시 충돌. SQLite 전환 필수.
- 캐시 폭증: `.cache/` 30일 TTL 있으나 일 1,000건 × 평균 5KB = 150MB/월. 6개월 누적 1GB. PC 사양에 따라 부담.
- 응답 지연 한도: 사서 인내 ≈ 5초. ISBN 조회는 캐시 시 0.5초, 미스 시 NL Korea 2~4초 OK. **Vision 사진은 10~20초** → 진행률 표시 필수 (현재 있는지 확인).
- 메모리 누수: pymarc Record 객체 다수 생성 후 미해제 가능. uvicorn worker 주기적 재시작 권장.

### F. 보안

- **API 키 유출 (사서가 SNS·블로그 게시)**: 현재 키 재발급 절차 없음. `/account/rotate-key` 엔드포인트 + 키 prefix(`kma_`) 기반 GitHub 검색 모니터링.
- 이메일 검증 없는 가입: `secrets.token_urlsafe(24)` 무한 생성 가능 → 매직 링크 OTP.
- **cloudflared 무료 도메인**: TryCloudflare URL은 매번 변경. **Named Tunnel + 사서가 본인 도메인 구매**가 안전.
- HTTPS·TLS: Cloudflare Tunnel 자동 처리 OK.
- **시크릿 마스킹**: `logging_config.py` 정상. 그러나 ANTHROPIC_API_KEY가 사서 키일 경우 BYOK이라 우리 책임 약함.

### G. 결제·청구·세무

- **사업자 등록**: 홈택스 5분, 무료. 업종코드 "응용 소프트웨어 개발 및 공급업".
- **간이과세자 (연 매출 1억400만 미만)** 유리. 매출 4,800만 미만 시 부가세 납부 면제.
- **통신판매업 신고 면제 조건**: 거래 50회 미만 또는 간이과세자. 우리는 **유료 가입 50회 도달 즉시 의무화** → 매출 발생 전 미리 신고 권장.
- **세금계산서**: 학교·공공 결제 필수. 토스페이먼츠 자동 발행.
- **카카오뱅크 직접 입금 vs PG**: 베타 단계는 직접 OK, 정식 결제 PG 필수 (분쟁·자동 갱신).
- **정기결제 자동갱신**: 약관 명시 + 이용자 해지 절차 1단계 (전자상거래법).
- **원천세**: 1인 사업자 본인 인건비 X. 외주 발주 시 8.8% 원천징수.

- **백신·방화벽**: V3·알약·시만텍이 cloudflared exe 차단 사례 다수. 화이트리스트 등록 가이드 필요.

### M. 사용자 교육

- **첫 30분 시나리오**: 가입 → 키 받기 → ISBN 1건 입력 → KOLAS 반입 검증. **영상 가이드 5분짜리 1개** 제작 권장 (사서 이탈 방지 결정적).
- 흔한 실수:
  - ISBN 13자리/10자리 혼동 → 자동 변환 OK
  - 저자명 오타 → 알라딘 폴백으로 보정
  - KDC 무지 → AI 추천 + KDC 트리 시각화 (`kdc_tree.py`)
  - 880 한자 누락 → hanja 라이브러리 자동
- **영상**: 유튜브 비공개 + 가입 이메일에 링크.

### N. 모바일 특수 이슈

- **데이터 사용량**: ISBN 1건 ≈ 10KB, 사진 1장 ≈ 500KB. 사서 폰 LTE 50건/일 = 25MB → 부담 적음. 사진은 1568px 자동 리사이즈로 50%.
- **카메라 권한**: Streamlit `st.camera_input` HTTPS 필수 (Cloudflare Tunnel 충족).
- **저장공간**: 웹앱이라 폰에 안 남음. 다만 사진 임시 캐시 브라우저 자동 정리 의존.
- **앱 아닌 한계**: 푸시 알림 X, 오프라인 X. PWA 전환 시 설치형 가능.
