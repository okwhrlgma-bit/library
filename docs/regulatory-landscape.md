# 규제·법률 환경 (한국 SaaS 운영용)

> 출시 전 PO가 1회 이해해야 하는 법적 환경 요약.
> 변호사 검수는 별도 (20~50만원, 매출 발생 후 권장).

---

## 1. 개인정보보호법

### 적용 영역
- 사서 이메일·도서관명 수집 → 개인정보 처리
- API 키 (사서 식별자)
- 사용량 로그

### 의무
- 개인정보처리방침 의무 게시
- 최소 수집 원칙 (꼭 필요한 정보만)
- 보관 기간 명시
- 제3자 제공 동의

### 우리 매칭
- ✅ `docs/terms-beta.md` 베타 약관 (개인정보 항목 4단)
- ✅ 키 해싱 (`server/usage.py` SHA256)
- ✅ 시크릿 마스킹 로깅 (`logging_config.py`)
- ⏸ 정식 개인정보처리방침 (출시 전 변호사 검수)

### 위반 시
- 과태료 천만원대까지

---

## 2. 정보통신망법

### 적용
- 통신판매업 신고
- 이용자 약관 명시 의무

### 우리 매칭
- ✅ 베타 약관 (`docs/terms-beta.md`)
- ⏸ 통신판매업 신고 (PO 직접, 구청)

---

## 3. 도서관법

### 우리 영향
- 도서관 자료(서지) 활용은 합법 (사실의 정보)
- KOLAS·DLS 데이터를 직접 가져오는 게 아니라 사서가 입력 → 도서관법 직접 영향 X

### 도서관 정책 위반 가능성
- 도서관별 데이터 외부 노출 정책 다름
- 우리 BYOK 모델은 사서 본인 통제라 안전

---

## 4. 저작권법

### 적용
- KORMARC 데이터 자체 = 사실의 정보 → **저작권 없음**
- KDC 분류표 = KLA 보유 (우리는 주류·강목까지만 무료 공개분 사용)
- DDC 분류표 = OCLC 보유, 라이센스 필요 (우리 미사용)
- 알라딘 데이터 = 출처 표시 의무 (계약상)
- 도서 표지 이미지 = 출판사·저자 저작권 (우리는 이미지 저장 X, 메타만)

### 우리 매칭
- ✅ 알라딘 출처 자동 표시 (`api/aladin.py`, `attribution`)
- ✅ KDC 트리는 무료 공개 부분 (주류·강목)
- ✅ 표지 이미지 URL만 저장, 다운로드 X
- ⏸ DDC는 글로벌 진출 시 라이센스 협상 (Phase 3)

---

## 5. 공정거래법 (B2B 약관)

### 적용
- SaaS 약관·환불정책
- 약관의 규제에 관한 법률

### 의무
- 일방적 불리 조항 금지
- 약관 변경 시 사전 통지

### 우리 매칭
- ✅ 베타 약관에 환불·해지 명시
- ⏸ 정식 약관 (변호사 검수)

---

## 6. 세법 (사업자)

### 적용
- 부가세 (간이과세자: 1억 미만 → 부담 적음)
- 종합소득세 (개인사업자) / 법인세 (법인)
- 세금계산서 발행 의무 (B2B)

### 우리 매칭
- ⏸ 사업자 등록 (PO 직접, 홈택스 5분)
- ⏸ 결제 시스템 세금계산서 자동 (토스페이먼츠 연계)
- ⏸ 자동 회계 도구 (자비스앤빌런즈 등, 월 1~3만원)

---

## 7. 공공기관 거래 (학교·시립도서관)

### 적용
- 나라장터 등록 (수의계약 한도 2천만원)
- 세금계산서 발행 필수
- 공공기관 정보보호 평가 (대형 SaaS는 ISMS 등)

### 우리 매칭
- ⏸ 나라장터 등록 (Phase 2 권장, 사서 1명 베타 후)
- ⏸ ISMS는 대형 단계에서

---

## 8. 약관 (출시 전 작성·검수)

### 필수 3종
1. **이용약관** — 서비스 이용 조건
2. **개인정보처리방침** — 개인정보 수집·이용
3. **환불정책** — 결제·해지·환불

### 작성 방법
1. 비슷한 SaaS 약관 참고 (Notion·Slack 한국어판)
2. AI Hub 약관 자동생성 활용
3. 변호사 검수 (20~50만원, 매출 발생 후)

### 우리 진행
- ✅ `docs/terms-beta.md` 베타 약관 초안
- ⏸ 정식 3종 + 변호사 검수

---

## 9. 약관 핵심 조항 (체크리스트)

### 책임 한계
- 데이터 손실 보상 한도 명시
- 외부 API 장애 면책
- 베타 기간 면책 강화

### 결제·환불
- 정기결제 자동 갱신 명시
- 해지 절차·환불 기준
- 분쟁 시 관할 법원

### 데이터 처리
- 보관 기간
- 백업·복구 정책
- 사용자 본인 데이터 다운로드 권리

### 우리 약관 보강 필요
- ⏸ "외부 API 변경으로 정확도 변동 가능" 명시
- ⏸ "AI 추천은 사서 검토 필수, 잘못된 분류 책임은 사서" 명시 (이미 있음, 강조)

---

## 10. 위반·분쟁 대응 (사전 준비)

### 가장 흔한 분쟁
- **데이터 손실** → 백업 자동화 + 약관 한도
- **결제 분쟁** → 환불 정책 명확
- **개인정보 누출** → 시크릿 마스킹·암호화
- **약관 미고지** → 가입 시 동의 화면

### 우리 매칭
- ✅ 백업 (`scripts/backup_logs.py`)
- ✅ 환불 정책 (`docs/terms-beta.md`)
- ✅ 시크릿 마스킹 (`logging_config.py`)
- ⏸ 가입 동의 화면 (Streamlit·랜딩 추가 가능)
