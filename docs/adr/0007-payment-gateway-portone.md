# ADR 0007 — 결제 PG (포트원 vs 토스페이먼츠 vs 카카오페이 직접)

**상태**: proposed (PO 사업자 등록 후 accepted 전환)
**일자**: 2026-04-26
**의사결정자**: PO

## 컨텍스트

베타 50건 무료 + 권당 100원·월정액 4단계 가격 모델. 매출 발생 시 결제 PG 필수.

운영 감사 단기 #5: "현재는 계좌이체/카카오페이 PO 직접 입금 → 학교/공공 결제팀 거부. PG 도입 + 자동 세금계산서 (사업자 등록 후 1주)".

옵션:
1. **포트원 (구 아임포트)** — 한국 1위 PG 어그리게이터. 토스·카카오·네이버페이·신용카드 한 번에
2. **토스페이먼츠 직접** — Stripe 비슷한 개발자 경험. 토스만
3. **카카오페이 직접 + 계좌이체 (현재)** — PG 비용 0, 학교 결제팀 거부 가능
4. **Stripe + 한국 어드민 변환** — 글로벌 동시 대응, 한국 결제팀 신뢰도 낮음

## 결정 (proposed)

**포트원 채택** — 사업자 등록 직후 도입. 토스페이먼츠는 백업.

근거:
- 토스·카카오·네이버페이·신용카드 어그리게이션 → 사서 친숙한 결제수단 모두 커버
- API 한 번 통합 = 향후 PG 변경 비용 낮음
- 한국 사업자번호 자동 검증·세금계산서 자동 발행 (학교·공공 결제팀 요구)
- PG 수수료 신용카드 2.7% / 간편결제 2.5~3.0% (업계 평균)

## 결과 (도입 후)

- 학교·공공도서관 결제팀 거부 0%
- 매출 인식 즉시 (사용량 → 청구서 → PG 결제 → 입금 자동)
- 사업자 등록·세무 의무 자동 충족 (포트원이 부가세 신고 양식 제공)

## 트레이드오프

✅ **장점**
- 사서가 토스·카카오·신용카드 중 익숙한 수단 선택
- 결제 실패 시 자동 재시도·환불 처리
- 정기결제(월정액) 자동 — 매월 PO 수동 입금 확인 불필요
- API 1개 = 백엔드 코드 단일

❌ **단점**
- PG 수수료 2.5~3.0% 고정 → 권당 100원 → 약 3원/건 차감 → 마진 90%
- 사업자 등록·통신판매 신고 선행 필수 (PO 1주 소요)
- 결제 실패 디버깅 시 PG 콘솔·우리 로그 양쪽 봐야

## 비용 시뮬

| 매출 | PG 수수료 (3%) | 우리 운영비 (5원/건) | 마진 |
|---|---|---|---|
| 월 30만원 (작은도서관 10관) | 9,000원 | 1,500원 | 92% |
| 월 200만원 (캐시카우 1단계) | 60,000원 | 10,000원 | 91% |
| 월 660만원 (캐시카우 3단계, 200관) | 198,000원 | 33,000원 | 90% |

여전히 마진 90%+ 유지. PG 수수료는 학교·공공 결제팀 신뢰 확보 비용으로 충분히 가치 있음.

## 완화 조치

- 도입 전: `docs/business-checklist.md` "사업자 등록 + 통신판매 신고" 우선 처리
- 도입 직후 1개월: 카카오페이 직접 + PG 병행 → 사서 사용 패턴 관찰
- 베타 사서 50명 도달 후: 포트원 단독 운영
- B2B 도서납품업체는 세금계산서 + 계좌이체 옵션 별도 유지 (PG 거치지 않음)

## 6개월 후 되돌릴 수 있는가?

**Y** — 포트원 → 토스페이먼츠 또는 Stripe 전환은 백엔드 결제 모듈 1개 교체. 결제 데이터는 우리 DB(`logs/billing/`)에 영속 저장 → 어떤 PG 사용해도 매출 추적 가능.
