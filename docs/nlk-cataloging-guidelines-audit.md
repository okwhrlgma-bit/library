## 3. 전거데이터 기술 지침 — 개인명·단체명 (2018, 각 57p)

| 항목 | 정보 |
|---|---|
| 배경 | KCR4판 = 표목 대신 접근점 사용·기본표목 미규정 |
| 적용 | KORMARC 통합서지용·전거통제용·소장정보용 |
| 표준 | KORMARC 전거통제용 2016년 개정 |
| 영역 | 개인명 (KORMARC 100·700) + 단체명 (KORMARC 110·111·710·711) |

### 자관 자체 작성 영역

> "전거데이터를 위한 국가 표준 규칙이 없기 때문에 KCR4판이 개정될 때까지 전거데이터를 일관성 있게 구축하고 기술하고자 개인명과 단체명을 위한 기술 지침을 마련"

→ 각 도서관마다 전거데이터 기술 차이 = 우리 SaaS 통일 가치.

### 우리 SaaS 적용

```python
# kormarc/authority.py (신규 후보)
class AuthorityRecord:
    """KORMARC 전거통제용 (KS X 6006-2)."""
    def create_personal_name(self, name, birth, death, ...): ...  # 100
    def create_corporate_name(self, name, place, ...): ...        # 110
    def create_meeting_name(self, name, year, place, ...): ...    # 111
    def link_bibliographic_record(self, kormarc_record): ...      # 700·710·711 부출
```

**우리 영역 X**: 전거레코드 자체 운영 X (NLK 전거 178만 활용만).
