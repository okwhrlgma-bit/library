## 0. 사서 워크플로우 전체 흐름

알파스 매뉴얼이 명세하는 단행본 처리 7단계:

```
[1단계] 이용자 신청 (홈페이지·모바일)
   ↓
[2단계] 희망자료관리 (§3) — 신청 검토·구입·기증 분기
   ↓
[3단계] 구입자료관리 (§4) / 기증자료관리 (§5)
   ↓ 주문·납품
[4단계] 검수자료관리 (§4) — 납품/미납 확인 → 등록인계
   ↓
[5단계] 등록자료관리 (§6) — 원부 부여 + 등록번호
   ↓
[6단계] 정리 (§7) — 목록완성·MARC 입력·청구기호·라벨·바코드   ← 우리 SaaS 핵심 영역
   ↓
[7단계] 배가일자기록 → 배가완료 (§8) → 이용자 검색·대출
```

**우리 SaaS 위치**: §7만 직접 지원. **§3·§4·§5·§6은 사서 시간의 60%+** 예상 차지하는데 우리 미흡수.

## 1. §3 수서대상관리 — 희망자료 (p.7~16)

### 알파스 워크플로우

| 단계 | 내용 |
|---|---|
| 입력 | 신청자(이용자) 검색 → 자료반입 (서명·저자·발행자·ISBN 검색) → 신청내용 작성 |
| **유형 옵션** | 우선대출예약·SMS수신·우선정리자료(★ 아이콘) |
| 검색 조건 | 비치상태(적용안함/신청중/처리중/소장중/취소)·신청일자·신청자 |
| 처리 분기 | 구입처리 → §4 / 기증처리 → §5 |
| 출력 | 검색 결과 Excel 출력 (모든 조건 일치 자료) |

### 우리 흡수 상태

| 항목 | 상태 |
|---|---|
| 희망자료 KDC 균형·예상 비용 분석 | ✅ `acquisition/wishlist.py` |
| 자관 중복 검사 | ✅ `acquisition/wishlist.py` |
| 신청자 정보·우선대출예약·SMS수신·우선정리자료(★) | ❌ 미흡수 |
| 비치상태 5종 (신청중·처리중·소장중·취소·적용안함) 라이프사이클 | ❌ 미흡수 |
| ISBN 검색 → 자료반입 | 🟡 부분 (`api/aggregator.py` 있지만 wishlist와 통합 X) |

### 신규 모듈 후보

```
acquisition/wishlist_lifecycle.py
  - WishlistStatus enum (PENDING·PROCESSING·OWNED·CANCELLED)
  - 우선대출예약·SMS수신·우선정리자료 플래그
  - 자료반입 자동 (api/aggregator → wishlist 일괄 매핑)
```

**평가축**: §0 +2분 (희망자료 1건 처리 시간) · §12 +1 (작은도서관 매월 5~20건 처리)

## 2. §4 구입 — 구입자료·검수·출력 (p.17~34)

### 알파스 워크플로우

#### 4.1 구입자료관리

| 단계 | 내용 |
|---|---|
| 입력 | 형식구분·서명·저자 (필수) + 자료반입 (ISBN 검색) |
| 구입정보 | 구입처·화폐구분·환율·할인율(%)·책수·비고 |
| 코드변경 | 구분코드 일괄 변경 (종·권 단위) |
| **차수관리** | 차수번호 = 발주 단위. 주문일자·검수일자·자료구분 |
| 주문번호 부여 | 차수 내 자료에 주문번호 자동 부여 |
| 주문처리 | 차수 단위 발주 완료 (책수·가격 합계 확인) |
| 출력 | 구입예정자료 Excel 반출 |

#### 4.2 검수자료관리

| 작업 | 설명 |
|---|---|
| 납품/미납 처리 | 권 단위 (종 일괄 X) — 검수 시 각 권 개별 |
| 주문취소 | 검수 단계에서 주문 취소 → 구입자료관리로 복귀 |
| 등록인계 | 납품 완료 → 등록자료관리(§6)로 이동 |

#### 4.3 구입자료출력

차수번호로 구입자료목록 출력 (Excel)

### 우리 흡수 상태

**모두 ❌ 미흡수**. 우리 `acquisition/wishlist.py`는 §3 희망자료 분석만 처리. §4 구입·검수는 0.

### 신규 모듈 후보

```
acquisition/purchase_order.py
  - PurchaseOrder (차수·자료·구입처·환율·할인율)
  - 주문번호 부여 (자동 시퀀스)
  - 주문처리 (차수 단위 발주 완료)
  - Excel 반출 (도서관 회계·내부 결재용)

acquisition/inspection.py
  - 권 단위 납품/미납 처리
  - 주문취소 (구입자료관리로 복귀)
  - 등록인계 (→ §6 등록자료관리)

acquisition/purchase_export.py
  - 차수번호 별 Excel/HCell 반출
```

**평가축**: §0 +5분 (구입 1차수 처리, 평균 50~100권) · §12 +2 (월 1회 차수 — 캐시카우 직결)

### 외부 시스템 연계

- **알파스**: 차수·접수번호 단위 관리. 우리 PurchaseOrder 모델 정합
- **KOLAS III**: 차수 개념 동일 (KOLAS 매뉴얼 정합 검증 필요 — C-3 후속 작업)
- **DLS (학교)**: 학교 회계 시스템 연계 별도 — 미조사

## 3. §5 기증 — 기증자료·기증자관리·출력 (p.35~55)

### 알파스 워크플로우

#### 5.1 기증자료관리

| 탭 | 입력 항목 |
|---|---|
| 기본정보 | 서명·저자·발행자·발행년·낱권ISBN·가격·편권차·권차서명·판사항·면장수·총책수 |
| 상세정보 | + 대등서명·총서명·발행지·세트ISBN·물리적 특성·크기·딸림자료 |
| 관리정보 | 우선정리자료·매체구분·보조등록·이용대상·이용제한 |
| **기증정보** | **기증자·기증일** (필수) |
| 책정보 | 대표책 선정·등록구분·일련번호·별치기호·매체구분 |
| 마크보기 | 생성된 KORMARC 확인 |

#### 5.2 기증자관리 (별도 메뉴)

기증자 정보 관리 (개인·단체·소속·연락처). 등록 → 기증자료에 자동 연계.

### 핵심: 기증접수번호 = 구입 차수 등가

기증자료는 접수번호로 묶음 처리. 구입의 차수와 동일 개념.

### 우리 흡수 상태

**모두 ❌ 미흡수**. 우리는 기증 워크플로우 전혀 없음.

### 신규 모듈 후보

```
acquisition/donation.py
  - Donation (서지·기증자·기증일·접수번호)
  - DonationReceipt (기증자별 누적 — 세금 영수증·감사장)
  - 기증자료 KORMARC 자동 (서명·저자 → 245)
  - 등록인계 (→ §6 등록자료관리)

acquisition/donor_management.py
  - Donor (개인·단체·연락처·기증 이력)
  - 기증자별 통계 (누적 권수·감사 발송)
```

**평가축**:
- §0 +3분 (기증 1건 처리 시간 — 마크 입력 + 기증자 등록)
- §12 +1.5 (연 100~500건 기증 받는 도서관 다수 — 작은도서관 특히)
- 부가 가치: **기증 영수증 PDF 자동 생성** = 작은도서관 자원봉사·CSR 도서관 핵심 기능

### 한국 기증 도서관 특수 사례

- 작은도서관 60% 이상이 기증 도서로 운영
- 학교도서관도 학부모회·교사회 기증 자료 빈도 높음
- 기증자가 "내가 기증한 책이 어떻게 활용되는지 보고 싶다" → 우리 SaaS에 기증자 포털 + 기증 통계 노출 = 차별점

## 7. 외부 시스템 연계 분석

### 알파스 vs KOLAS III vs 우리 SaaS

| 단계 | 알파스 | KOLAS III | 우리 SaaS | 우리 영역 |
|---|---|---|---|---|
| §3 희망자료 | ✅ 자체 메뉴 | ✅ 자체 메뉴 | 🟡 부분 | 사서 자관 도구 |
| §4 구입 | ✅ 차수 관리 | ✅ 차수 관리 | ❌ | 도서관 회계와 연계 — 우리 직접 X. **우리 = MARC 생성에 집중** |
| §5 기증 | ✅ 접수번호 | ✅ 접수번호 | ❌ | 영수증·감사장 = 우리 신규 모듈 가치 |
| §6 등록 | ✅ 원부 | ✅ 원부 | 🟡 부분 (번호만) | 통계 자동화 = 우리 가치 |
| §7 정리 | ✅ MARC 입력 | ✅ MARC 입력 | ✅ **우리 핵심** | KORMARC AI 자동 생성 = 우리 차별점 |

**전략**: §4·§6은 **알파스·KOLAS와 정면 경쟁 X** (도서관 회계 통합·원부 의무라 ILS 본체 영역). 대신 §3·§5·§7은 **사서 시간 단축 + AI 가치** 영역으로 우리가 우위 가능.

### 우리 우위 영역 (§3·§5·§7)

| 영역 | 알파스/KOLAS 한계 | 우리 가치 |
|---|---|---|
| §3 희망자료 분석 | 사서가 KDC 균형·중복 수동 검토 | **AI 자동 KDC 분포·중복 검사 + 예상 비용** ✅ |
| §5 기증 영수증·감사장 | PDF 출력 X (Excel만) | **PDF 자동 생성 + 기증자 포털** (우리 신규) |
| §7 KORMARC AI 자동 | 사서 수동 입력 (45+ 필드) | **사진 1장·ISBN → 5초 자동 + 사서 검수** ✅ |
