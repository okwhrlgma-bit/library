# kormarc-auto 가격 모델 (MVP-1)

## 목표

**사서가 자기 예산으로 결제할 만한 가격 + 우리(PO)는 비용 + 마진**

### 캐시카우 도달 경로

| 단계 | 고객 수 | 월 매출 | 비고 |
|---|---|---|---|
| Phase 1 (~12개월) | 작은도서관 50곳 × 3만원 | **150만원** | 운영비 커버 시작 |
| Phase 2 (~24개월) | 100곳 (소·중 섞임) | 약 400만원 | PO 인건비 일부 |
| Phase 3 (~36개월) | 200곳 | **600만원** ★ BEP | 본격 캐시카우 |

**진짜 자동에 가까운 구간**: Phase 3 이후. 정기결제·낮은 이탈률·입소문 영업.

## 광고 수익 모델 — 비채택

검토 결과: 도서관 트래픽은 광고 단가가 낮고 사서 거부감 크며, 운영비를 못 메움.
- 도서관계 CPM 1,000~3,000원
- 사서 1,000명 기준 월 광고 수익 90~270만원
- 운영비(서버·DB·도메인) 월 50~135만원 → 마진 거의 없음
- 학교도서관·공공기관은 광고 차단 정책 있는 곳도 다수

→ **구독 SaaS 단일 모델**로 집중. 광고 배너·전면광고 절대 추가 안 함.

## 비용 구조 (PO 입장)

| 항목 | 권당 비용 |
|---|---|
| Anthropic Vision (Sonnet) | 약 1.5원 (캐시 미스 시) |
| Anthropic Vision (Haiku, ISBN만) | 약 0.1원 |
| Anthropic KDC AI (Sonnet, prompt cached) | 약 0.3원 |
| 외부 API (NL Korea/알라딘/카카오) | 0원 (무료 한도 내) |
| Cloudflare Tunnel | 0원 |
| 서버 호스팅 (Fly.io 등) | 약 5원/건 (월 만원 분담) |
| **합계** | **약 7원/건** |

권당 100원이면 **마진 ≈ 93%**. 1,000건/월이면 사서 5만원 결제 → PO 4.6만원 이익.

## 결제 인프라 (단계)

| 단계 | 시점 | 도구 |
|---|---|---|
| MVP-1 (현재) | 베타 무료, 가격 표시만 | 정적 안내 페이지 (`PAYMENT_INFO_URL`) |
| MVP-2 | 사서 5명 베타 검증 후 | 카카오페이 송금 + 수기 입금 확인 |
| MVP-3 | 누적 매출 100만원 달성 | 포트원/토스 정식 결제, 자동 영수증 |
| MVP-4 | 도서관 10관 도입 | 사업자 등록, 세금계산서 발행 |

## 차별화 (vs 북이즈·제이넷)

| 항목 | 기존 | 우리 |
|---|---|---|
| 가격 | 도서관 1관 연 수백만~수천만원 | 권당 100원 / 월 5만원~ |
| 도입 | 시스템 전체 교체 | API 호출만 — 기존 KOLAS와 병행 |
| 사용 매체 | PC 전용 | 폰 브라우저로도 |
| 옛 책·기증도서 | 사서 수기 | Vision 자동 |
| 셋업 시간 | 컨설팅 1~2주 | 5분 (가입 + API 키) |

## 베타 모집 ICP

- 1인 사서 학교도서관 (가장 시간 압박, 전체 마크 부담)
- 신규 개관 작은도서관 (KOLAS는 이미 있는데 마크 인력 없음)
- 출판사 자가 KORMARC 생성 (기증·납본 시)
