## 2. ⚠️ 미흡수·부분 흡수 워크플로우 12건 (신규 commit 후보)

### 2.1 종통합·종분리 (p.97~99) — ❌ 미흡수

**알파스 워크플로우**:
- **종통합**: 2개 이상의 종자료 → 1개로 통합. 대상자료의 505·049 Tag가 통합 주자료로 이동
- **종분리**: 다권본 1개 → 2개 종으로 분리. 권차수만큼 일괄 분리

**우리 미흡수**: 우리 SaaS는 **단방향 생성 (ISBN → KORMARC)** 만 지원. 통합·분리는 사후 편집 영역.

**신규 모듈 후보**: `kormarc/record_merge.py` (record_merge·record_split 함수)
**평가축 영향**: §0 사서 마크 시간 +0.5분 추가 단축 (다권본 처리 시)
**우선순위**: 중 (다권본 ICP인 도서관에 가치)

### 2.2 복본추기·복본해제 (p.79~80) — 🟡 부분 흡수

**알파스 워크플로우**:
- 자관 입력된 복본자료 자동 검색
- 복본추기 → 049 ▾l (등록번호) 묶음 추가 (예: `▾l 1234▾l 1235▾l 1236`)
- 복본해제 → 묶음 해제

**우리 부분 흡수**: 049 ▾c (복본기호 c.1·c.2) 만 처리. ▾l 등록번호 묶음 X.

**신규 모듈 후보**: `librarian_helpers/duplicate_copy.py` (find_duplicates·attach_copy_to_existing 함수)
**평가축 영향**: §0 사서 시간 +1분 단축 (수동 검색 → 자동)
**우선순위**: 높음 (모든 도서관 매주 복본 처리 발생)

### 2.3 마크반입 (Excel·HCell·MARC 일괄) (p.92~96) — ❌ 미흡수

**알파스 워크플로우**:
- 반입 형태 3종: Excel·HCell(한컴 셀)·MARC 파일
- 정상자료 / 오류자료 / 성공자료 3 탭 분리
- 오류자료 마크보기 화면에서 수정 → 정상 탭으로 복귀
- 정리중반입 (정리대상 상태) vs 소장자료반입 (즉시 비치)
- 원부적용 (원부번호 부여) 옵션

**우리 미흡수**: 우리 SaaS는 ISBN 단일 또는 일괄 (텍스트) 처리만. Excel·HCell 직접 반입 X.

**신규 모듈 후보**: `import/marc_importer.py` + `import/excel_importer.py` + `import/hcell_importer.py`
**평가축 영향**: §0 시간 +5분 단축 (대량 반입 시)
**우선순위**: 매우 높음 (도서관자동화 종합 대시보드 §0 KPI 핵심)

### 2.4 공동목록반입 (Kolis-net 검색 후 반입) (p.86) — 🟡 부분 흡수

**알파스 워크플로우**:
- Kolis-net (국립중앙도서관·공공도서관 종합목록) 검색
- 검색 결과 서지 정보 반입
- 자관 KORMARC에 자동 매핑

**우리 부분 흡수**: `api/kolisnet_compare.py` 비교만. 자동 반입 X.

**신규 모듈 후보**: `import/kolisnet_import.py` (search·fetch·convert_to_local 함수)
**평가축 영향**: §0 시간 +3분 단축 (수동 검색 → 자동)
**우선순위**: 높음 (KOLAS III 마이그·신규 자료 모두에 활용)

### 2.5 006 고정길이 필드 (008 보완) (p.86) — ❌ 미흡수

**알파스 명세**:
> "008 필드의 보완 필드로 입력대상자료에 다른 형태의 자료(특성)가 포함되어 있거나 딸림자료가 있는 경우, 이에 관한 부호화정보(008 필드에 해당)를 추가로 기입"

**우리 코드 추가**: `kormarc/builder.py` `build_006_for_companion` 함수 신규

## 3. binary_assertions 추가 후보

알파스 매뉴얼 p.74 명시:
> "008Tag(부호화정보필드), 090Tag(자관 청구기호), 245Tag(표제와 책임표시사항), 260Tag(발행, 배포, 간사 사항), 300Tag(형태사항), 653Tag(비통제 색인어)는 필수입력항목으로 미입력 시 저장이 불가능합니다."

우리 `binary_assertions.py` 현재 008·245만 명시. 추가 후보 4건:

| 어셔션 후보 | 검증 |
|---|---|
| `assert_kormarc_090_present` | 자관 청구기호 필수 |
| `assert_kormarc_260_present` | 발행·배포·간사 필수 |
| `assert_kormarc_300_present` | 형태사항 필수 |
| `assert_kormarc_653_present` | 비통제 색인어 필수 |

**우선순위**: 매우 높음 (사서 검수 시 6개 필수 필드 자동 검증 = 신뢰성 ↑ + KORMARC 호환성 보장)
