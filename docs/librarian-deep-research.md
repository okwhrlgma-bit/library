## 2. Aycock (2025) 사례연구 — 학술 검증

### 2.1 실험 (텍사스 주립대학)
- 2024.12, CatalogerGPT (OpenAI 플러그인)
- 100권 전자책 학술회의 논문집 MARC 자동 생성
- 25회+ 프롬프트 반복

### 2.2 발견된 오류 종류
| 오류 유형 | 예 |
|---|---|
| **ISBN 축약** | "978-0-7918-8684-7" → "978079188" |
| **표제 부정확 전사** | 환각 |
| **부분필드 오류** | $r 대신 $c 사용 |
| **표목 오류** | $d → $c 잘못 |
| **목차·저자 환각** | 검증 부담 ↑ |

### 2.3 결론
> "생성형 AI는 초안 작성에 유용하나 사서가 감시하에서만 작동"

수동 15분/9권 vs AI 작성 후 검수 부담 큼.

### 2.4 권고 (Aycock)
1. 중요 필드를 프롬프트에서 강조
2. 모델 레코드 작성 후 활용
3. AI 도구 제안 프롬프트 참고
4. 협력 목록작업 DB 제출 전 철저 검수

### 2.5 우리 도구 매칭
- ✅ tool_use 강제 (ISBN·식별기호 오류 방지)
- ✅ 후보 형태로 제시 (사서 검수 강제)
- ✅ ISBN 체크섬 검증
- ✅ NL Korea·정보나루 1순위 (AI 단독 X)

→ Aycock 권고 4종 모두 우리 코드 반영. 학술적 정당성 확보.

## 4. 정보나루 API 활용 가이드 (data4library.kr)

### 4.1 제공 API 약 15종
- 도서관 조회
- 장서/대출 데이터
- 인기도서
- 추천도서 (마니아용·다독자용)
- 도서 상세정보 (ISBN)
- 키워드 분석
- 이용분석
- 급상승도서 (7일 추이)

### 4.2 인증·호출
- 회원가입 → 인증키 신청 → 발급
- 모든 호출 `authKey` 파라미터 필수
- 응답: **XML** 형식
- 페이지네이션 (`pageNo`, `pageSize`)

### 4.3 우리 매칭
- ✅ `api/data4library.py` (키워드만 사용 중)
- ⏸ 인기도서·추천도서·이용분석 = **자관 통계 보고서 강화 가능**
- ⏸ 급상승도서 = **사서 수서 추천 기능 추가 가능**
