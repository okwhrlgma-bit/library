# 글로벌 진출 단계 전략

> 한국 → 일본 → 영어권 → 중남미·동남아 순. "동시 5개국 진출" 금지.

---

## Phase 1 (현재~24개월): 한국 단독

- **타겟**: 공공·학교·작은도서관 + 출판사 자가 KORMARC
- **기술**: KORMARC + KDC + KCR4 (현재 v0.3 동작)
- **검증**: 사서 5명 → 50개 도서관 → 매출 월 500만원
- **핵심 자산**: 한국 사서 평판·레퍼런스 도서관 30~50곳

전제: 이 단계 완료 전 해외 진출 금지. 한국에서 신뢰 안 쌓이면 해외도 안 됨.

---

## Phase 2 (24~36개월): 일본 진출

**왜 일본 먼저?**
- 도서관 운영 구조 한국과 매우 유사 (정부 주도형)
- NACSIS-CAT (일본판 KOLIS-NET) 호환 가능
- 시차 없음 — 1인 응대 가능
- NDC ↔ KDC 매핑 비교적 단순

**필요 작업**
- 분류 시스템: NDC 활성 (`classification/scheme.py`의 placeholder 구현)
- API 어댑터: NDL Search API (국립국회도서관)
- 로마자화: 일본어 Hepburn (한국어 ALA-LC와 별도 라이브러리)
- UI 일본어 번역
- JAPAN/MARC 형식 출력 (MARC21 변형)

**예상 작업량**: 3~4개월 (분류 데이터 수집·라이센스 협상 포함)

---

## Phase 3 (36~48개월): 영어권 (미국 + 영국)

**대상 ICP**
- K-12 학교도서관 (저예산, 무료/저가 도구 갈증)
- 처치 라이브러리 (교회 도서관)
- 작은도서관·동네 도서관
- **NOT** 대학·연구도서관 (OCLC WorldCat 종속, 우리 진입 어려움)

**필요 작업**
- DDC 라이센스 협상 (OCLC WebDewey, 연 수천 USD)
- 또는 LCC만 활성 (LC 자체 데이터, 무료지만 양 많음)
- LC Catalog API 연동 (정보나루 대신)
- ALA-LC 로마자화 (한국어 → 영어)
- UI 영어 번역
- 시차 응대 — 현지 파트너십 또는 챗봇 보강

**리스크**: MarcAI.cloud 같은 기존 경쟁자 — 가격·UX로 차별화 필요.

---

## Phase 4 (48개월+): 중남미 + 동남아

- 스페인어·포르투갈어 UI
- UDC 분류 옵션 (라이센스 필요)
- 결제 인프라 현지화 (Stripe·PayPal)
- 로컬 파트너 필수 (시차·언어 응대)

---

## 처음부터 추상화 해두는 것 (코드 구조)

이번 세션에 **추상화 인프라 자리 마련 완료**:

1. ✅ **분류 시스템 추상화**: `classification/scheme.py` — KDC/DDC/NDC/LCC/UDC 인터페이스
2. ✅ **서지 DB 어댑터 패턴**: `api/aggregator.py` — NL/알라딘/카카오 분리, NDL·LC·BL 추가 가능 구조
3. ✅ **MARC 형식 변환**: `conversion/marc21.py` — KORMARC ↔ MARC21 양방향
4. ⏸ **i18n**: 미구현 — Phase 2 진입 시 streamlit-i18n 추가

**원칙**: Phase 2 활성 전까지 placeholder만 두고 구현 안 함. 한국에서 검증 안 끝났는데 해외 진출 코드 쓰면 모두 미숙성.

---

## 솔직한 우려 (재확인)

1. **동시 진출 욕심 금지** — 1인이 5개국 시장 펼치면 어디도 신뢰 못 얻음
2. **로컬라이징은 UI 번역이 아님** — 분류·서지DB·로마자화·문화 전체. 나라당 3~6개월
3. **고객지원 시차** — 미국 동부 응대는 새벽 근무. 챗봇으로는 부족, 사람 필요한 직군

**결론**: 한국에서 50개 도서관 + 월 500만원 달성 후 일본부터 1단계씩.
