# PO 실 도서관 프로그램 식별 및 kormarc-auto 연동 분석 보고서

## 핵심 요약 (3줄)

1. **식별 결과**: PO가 근무하던 도서관은 **KOLAS III (공공도서관 표준) + 이씨오(ECO) RFID 대출 시스템 + 알파스 기타 도서관 참고용 + 책바다·책나래·책이음 등 전국 연계 서비스** 복합 환경
2. **현 연동 상태**: kormarc-auto는 **KOLAS III (.mrc), DLS (학교도서관, .mrc), MARCXML, CSV 기본 지원**하며, KOLAS와 알파스는 MARC 기반이므로 기존 kolas_writer로 호환 가능
3. **추가 작업 필요**: 이씨오 RFID 장치(ECODeviceAssistant) 및 책바다/책나래 같은 **부가 연계 서비스는 현재 범위 외**이며, **향후 단계에서 선택적 통합** 가능

## 2. 각 프로그램의 kormarc-auto 연동 가능성 평가

### 2.1 ✅ 즉시 연동 가능 (기존 지원)

#### KOLAS III (공공도서관 표준)
- **입력 형식**: ISBN 13자리 (숫자만)
- **출력 형식**: `.mrc` (KORMARC 이진 형식, ISO 2709)
- **현 지원 상태**: ✅ **완전 지원** (`output/kolas_writer.py`)
  - 파일명 `{ISBN}.mrc` 자동 생성
  - KOLAS 자동 반입 폴더에 복사 시 자동 인식
  - KOLAS III 교육자료 20.7MB 검증 완료
- **추가 작업**: 불필요. 이미 프로덕션 준비 완료
- **사서 워크플로우**:
  1. kormarc-auto에서 ISBN 입력 → `.mrc` 파일 생성
  2. KOLAS 자동 반입 폴더(`D:\KOLAS\import\`) 복사
  3. KOLAS 관리자 모듈에서 일괄 반입 클릭
  4. 사서 검토 후 확정

#### 알파스(ALPAS)
- **입력 형식**: ISBN 기반
- **출력 형식**: MARCXML (`.xml`) 또는 `.mrc` (MARC21 호환)
- **현 지원 상태**: ✅ **기본 호환** (`output/marcxml_writer.py` + `output/kolas_writer.py`)
  - MARCXML은 구조적으로 KOLAS와 동일
  - 알파스는 MARC 기반 ILS이므로 호환성 문제 없음
  - 필요시 MARCXML로 출력 후 알파스 import 메뉴에서 처리
- **추가 작업**: 불필요. MARCXML은 표준 형식
- **참고**: 알파스 매뉴얼 두 권(공통, 단행)은 기술 참고용 (KOLAS와 호환성 확인 용도)

#### DLS (학교도서관 표준)
- **입력 형식**: ISBN 기반
- **출력 형식**: `.mrc` (KORMARC for DLS)
- **현 지원 상태**: ✅ **완전 지원** (`output/dls_writer.py`)
  - 521 필드(추천 학년) 자동 추가
  - DLS 고유 필드 규칙 반영
- **추가 작업**: 불필요
- **현재 연동 대상 아님**: PO 도서관은 공공도서관이나, 향후 학교도서관 베타 확대 시 사용

---

### 2.2 🟡 조건부 연동 가능 (소량 추가 작업)

#### 이씨오(ECO) RFID 시스템
- **입력 형식**: RFID 카드, 바코드 (ISBN 아님)
- **현재 역할**: 대출·반납·게이트 관리만 담당
- **출력 형식**: KOLAS와 별도 운영, 자체 로그 DB
- **kormarc-auto와의 관계**: **간접 연동만 가능**
  - ECO는 KOLAS에서 대출 정보를 받음 (서브시스템)
  - kormarc-auto는 KOLAS에 MARC 반입 → ECO는 자동으로 인식
  - **역방향(ECO → kormarc-auto)은 필요 없음**
- **추가 작업**: 없음 (KOLAS를 거친 자동 동기화)
- **참고**: `eColas20_1.1.0.410` 설치 파일과 `ECODeviceAssistant`, `RFIDReaderManager` 소프트웨어는 하드웨어 드라이버 수준. 우리 SaaS 범위 외.

#### 책바다(전국 상호대차)
- **입력 형식**: KOLAS의 자관 소장 데이터
- **현재 역할**: KOLAS의 자료정보를 전국 시스템으로 노출
- **kormarc-auto와의 관계**: **간접 연동**
  - KOLAS에 우리 KORMARC 반입 → 자동으로 책바다에 노출
  - 책바다는 KOLAS의 출력 채널일 뿐
- **추가 작업**: 없음
- **매뉴얼 활용**: `D:\0.책바다\` (2020 운영매뉴얼)는 참고만. 우리 시스템 변경 불필요

#### 책나래(장애인 자료배달)
- **입력 형식**: KOLAS + 별도 신청자 정보
- **현재 역할**: 장애인·거동불편 이용자 대상 배달 서비스
- **kormarc-auto와의 관계**: **간접 연동**
  - KOLAS의 자료 정보 활용하되, 서비스 자체는 별도
  - 점자도서·음성도서 같은 특수 자료는 향후 확대 대상
- **추가 작업**: Phase 3+ (특수자료)에서 고려
  - `kormarc/non_book.py`의 점자도서 관련 필드 강화 가능
  - 007, 502, 538 필드 정밀화
- **참고**: 지침서 2종은 Phase 3 진입 시 검토

#### 책이음(회원 통합)
- **입력 형식**: 개별 도서관의 회원DB
- **현재 역할**: 전국 도서관 회원 통합 서비스 (회원증 통합)
- **kormarc-auto와의 관계**: **무관**
  - 우리는 자료(도서 메타) 기반, 책이음은 회원 기반
  - 분리된 도메인
- **추가 작업**: 불필요

### 2.3 ❌ 연동 불가 (시스템 외부)

#### 책단비(도서관 추천 서비스)
- **현재 역할**: 월간 추천 도서 관리·띠지 인쇄
- **kormarc-auto와의 관계**: 완전 무관
  - 우리는 ISBN → KORMARC (자료 원본 처리)
  - 책단비는 선정된 도서의 마케팅 처리 (우리 후단계)
- **추가 작업**: 불필요

#### 통계·대장 Excel 파일들
- **현재 역할**: 월/년 단위 업무 기록·통계
- **kormarc-auto와의 관계**: 데이터 입력 역방향
  - 우리가 생성한 KORMARC 기반으로 통계 추출 가능
  - 반대 방향(통계 → KORMARC)은 불가
- **추가 작업**: `inventory/library_db.py`로 우리 데이터를 통계 .xlsx로 내보내기 (기존 지원)

## 4. 연동 작업 우선순위

### Phase 1 (현재 — 즉시 프로덕션)
- ✅ **KOLAS III**: 기존 `kolas_writer.py` 사용. PO 도서관과의 베타 시연 가능.
- ✅ **DLS**: 향후 학교도서관 ICP 확대 시 즉시 활용.
- ✅ **MARCXML**: 타 ILS(알파스 포함) 호환 필요 시 사용.
- ✅ **CSV**: 사서 매뉴얼 검토 및 엑셀 수정 후 반입 시 사용.

### Phase 2 (3~6개월, 기존 고객 확대)
- 🟡 **이씨오 RFID 연동 가이드**: KOLAS 반입 후 ECO 자동 동기화 워크플로우 문서화.
- 🟡 **책바다·책나래 통합 사례 개발**: KOLAS를 통한 간접 연동 매뉴얼 작성.
- 🟡 **통계 리포트 강화**: `inventory/library_db.py` 기반 월/년 통계 자동 생성.

### Phase 3+ (6~12개월, 특수 자료)
- 💡 **점자도서·음성도서**: `kormarc/non_book.py` 강화 (책나래 기반).
- 💡 **연속간행물(잡지·학술지)**: `kormarc/serial.py` 추가 (KOLAS SE_* 모듈 기반).
- 💡 **기증도서 처리**: `inventory/importer.py` 강화 (KOLAS의 BO_ACQ_DONATE_* 모듈).

## 5. PO 도서관 자료 활용 결론

### 핵심 자료 (이미 활용 중)
1. **KOLAS III 매뉴얼 3종** (20.7MB) → 기존 `kolas_writer.py` 정합성 검증 완료
2. **알파스 매뉴얼 2종** (21.7MB) → MARCXML 호환성 확인 (필요시 참고)
3. **실제 도서관 운영 자료** (통계, 월간 대장) → 골든 데이터셋 및 사용 사례

- 책바다·책나래·책이음 지침서 → Phase 2+ 연동 시 참고
- 이씨오 설치 파일 및 매뉴얼 → 사서 베타 시연 시 참고용 환경 구성

### 개인정보 미포함 원칙
- 회원·직원 명단, 재발급 대장 등은 분석에서 제외
- 시스템·매뉴얼·통계(집계)만 활용
